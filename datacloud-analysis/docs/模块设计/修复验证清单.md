### ✅ 8. Executor.run() 异步调用修复
**问题**: OqlRouter 同步调用 executor.run()，但该方法是 async 且需要 request_id 参数

**修复文件**:
- `packages/datacloud-data/src/datacloud_data_sdk/oql/router.py`
- `packages/datacloud-data/src/datacloud_data_sdk/oql/cross_source_executor.py`
- `packages/datacloud-data/src/datacloud_data_sdk/oql/pipeline_executor.py`
- `packages/datacloud-analysis/src/datacloud_analysis/tools/oql/query_objects.py`

**修复内容**:
1. **OqlRouter.route()** 改为 async，添加 request_id 参数
2. **OqlRouter.execute_single_step()** 改为 async，添加 request_id 参数
3. **CrossSourceExecutor.execute()** 改为 async，添加 request_id 参数
4. **CrossSourceExecutor._execute_cross_link()** 改为 async，添加 request_id 参数
5. **CrossSourceExecutor._fetch_sub_records_batched()** 改为 async，添加 request_id 参数
6. **PipelineExecutor.execute()** 改为 async，添加 request_id 参数
7. **query_objects()** 工具保持同步，使用 loop.run_until_complete() 包装异步调用

**关键修改**:
```python
# router.py line 143-145
step_results = await executor.run([task], request_id)
result = step_results.get_result(task.step_id)
return result if isinstance(result, list) else [result]

# cross_source_executor.py line 128-129
step_results = await executor.run([task], request_id)
main_records = step_results.get_result(task.step_id)

# cross_source_executor.py line 278-279
step_results = await executor.run([task], request_id)
result = step_results.get_result(task.step_id)

# query_objects.py - 同步工具函数，使用事件循环包装
import uuid
import asyncio
request_id = (
    ((config or {}).get("configurable") or {}).get("thread_id")
    or uuid.uuid4().hex[:12]
)

try:
    loop = asyncio.get_event_loop()
except RuntimeError:
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)

records = loop.run_until_complete(router.route(
    oql_params=oql_params,
    term_resolver=term_resolver,
    executor=executor,
    datasource_registry=datasource_registry,
    request_id=request_id,
))
```

**验证方法**:
```bash

python -m py_compile src/datacloud_data_sdk/oql/cross_source_executor.py
python -m py_compile src/datacloud_data_sdk/oql/pipeline_executor.py
python -m py_compile src/datacloud_analysis/tools/oql/query_objects.py

# 所有文件应该输出 "Syntax OK"
```

## 故障排查

### 问题 1: OqlRouter 未初始化
**症状**: 查询返回空结果，日志中无本体 Schema

**检查**:
```bash
grep "OqlRouter initialized" logs/worker.log
```

**解决**: 确保 bootstrap.setup() 被调用

### 问题 2: 数据源配置错误
**症状**: 工具调用失败，数据库连接错误

**检查**:
```python
from datacloud_analysis.dependencies import get_executor
executor = get_executor()
# 检查数据源配置
```

**解决**: 验证环境变量 DB_* 是否正确设置

### 问题 3: 本体文件未找到
**症状**: 知识注入中间件报错，找不到 OWL 文件

**检查**:
```bash
echo $DATACLOUD_ONTOLOGY_SCENE_PATH
ls -la $DATACLOUD_ONTOLOGY_SCENE_PATH/ontology/
```

**解决**: 确保环境变量指向正确路径

### 问题 4: 参数类型错误
**症状**: query_objects 调用失败，JSON 解析错误

**检查**:
```bash
grep "tool_args=" logs/worker.log | tail -1
```

**解决**: 确保 _parse_json_param() 正确处理 JSON 字符串

### 问题 5: Executor.run() 调用错误
**症状**: 出现 "missing 1 required positional argument: 'request_id'" 错误

**检查**:
```bash
grep "missing.*request_id" logs/worker.log
```

**解决**: 确保所有 executor.run() 调用都使用 await 并传入 request_id
