# 董老师主要要求和建议梳理如下：
一、总体时间节奏上--按330作为下一个时间节点。四个业务方向的优先级，将偷税漏税风险识别放到最后，第一做好企业的分层和呈现、第二把税收做好（呈现税收收入跟投入之间的供需配比关系）。

二、打法策略上--核心是数据能力的建设，要求先把MVP最小数据基石做扎实（基于高德、浩鲸以及可外采的数据），1.0版本的功能建设主要围绕这些数据开展。有三个要点，第一是把所有基石数据项补齐和做好每一项数据来源的审核。第二是把数据处理及数据维护更新做好，至少保证月更。第三是关注数据获取成本，前期将数据成本作为考察指标。

三、数据能力建设相关部分：
1、网格规划--包括基础网格（高德的100米*100米网格）、充分调研好用户业务定义的网格（把业务网格和业务管理的层次定义好），做好对应匹配关系，提升计算性能。参考董老师提供的基于网格的数据管理体系相关文档。
2、企业事实表--建议参考高德的划分标准（S、A、B、C、D、E）对企业做好分类，每一类企业对应不同业务的处理方式。
3、企业税收--基于AI能力做好推演，即通过可获取的数据（自有数据和外采数据），推测企业的年经营额和纳税额。建议一方面参考高德云图金融团队小微普惠企业做房贷的评估方案，另一方面针对上市公司获取其财务报表。
4、资产事实表--如何利用高德现有数据和已有的数据把亦庄资产事实表推导出来，资产至少以楼为单位，建议按楼块、园区、地块（产业园）三个维度形成，同时推演出政府对资产的定义是什么。
5、网格活跃度评价--针对google等卫星遥感数据，建议是采买付费账号验证其数据能力，明确其商业政策。同时希望是以在线接口调用方式（可看下阿里云和中科院等单位的相关产品）。
6、产业链--建议参考金融市场的行业划分类别，同时针对类似SAB的企业，做好不同行业的企业上下游供应链的分析。

四、系统演示相关部分：
1、登录模式--建议采用手机号验证登录。
2、产品名称--建议采用流行的claw类名称。
3、产品性能--问答响应时间不超过1秒，数据查询不超过100ms，基线是和豆包、千问等智能体保持相同水准，必要时高德可安排相关人员协助优化。
4、产品展示模式--参考豆包、千问等智能体产品展现形式，地图可随时被拉起。
