# 意图路由指南

## 意图分类

| 用户表达 | 意图 | 调用脚本 |
|----------|------|----------|
| 创建/新建/添加 + 对象/表 | 创建对象 | `create_object.py` |
| 修改/更新/编辑/添加字段 + 对象 | 修改对象 | `modify_object.py` |
| 删除/移除/清除 + 对象 | 删除对象 | `delete_object.py` |
| 创建/新建 + 视图/场景 | 创建视图 | `create_view.py` |
| 修改/更新 + 视图/场景 | 修改视图 | `modify_view.py` |
| 删除 + 视图/场景 | 删除视图 | `delete_view.py` |
| 查看/列出/有哪些 + 对象/视图 | 查询列表 | `list_resources.py` |

## 易混淆场景

| 用户说 | 正确意图 | 错误意图 |
|--------|---------|---------|
| "给客户对象加一个备注字段" | modify_object（新增字段） | create_object |
| "修改客户对象的字段" | modify_object | create_object |
| "删除客户对象的某个字段" | modify_object（字段删除需二次确认） | delete_object |
| "删除客户对象" | delete_object（整个对象，需二次确认） | modify_object |
| "创建一个视图包含客户信息" | create_view | create_object |

## 信息收集策略

### 创建对象时，LLM 应主动引导用户提供：

1. **对象名称**（必须）
2. **对象描述**（可选，可跳过）
3. **字段列表**（至少一个业务字段，id 主键自动注入）
4. 每个字段的：名称、数据类型、是否必填、属性角色（DIMENSION/MEASURE）
5. 对 DIMENSION 类型字段询问：是否有固定枚举值？
   - 有 → 收集枚举值列表（DICT_TERM）
   - 无 → 跳过
6. **可选**：关系（与哪些对象关联）
7. **可选**：自定义 Action

### 创建视图时，LLM 应主动引导用户提供：

1. **视图名称**（必须）
2. **视图描述**（可选）
3. **引用的对象列表**（至少一个）
4. **字段列表**（每个字段需指定来源对象和来源字段）
5. 每个字段的同义词（可选，用于 BI 自然语言理解）

## 二次确认场景

以下操作必须在执行前向用户二次确认：

- **删除对象**：提示"将同时删除 OWL 定义和 SQLite 表，数据不可恢复"
- **删除视图**：提示"将删除视图定义"
- **删除字段**：提示"字段删除后 SQLite 列数据将丢失"（本期不支持，提示用户）

## 取消操作

任何步骤前用户说"取消"/"退出"/"算了"，直接退出，不执行任何操作。
