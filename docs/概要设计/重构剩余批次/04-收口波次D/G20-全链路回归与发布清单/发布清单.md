# G20 发布清单

## 1. 发布前检查
1. [ ] `dev` 分支工作区干净，无未提交改动。
2. [ ] `回归报告.md` 已生成，必测项全部通过。
3. [ ] `风险清单.md` 已评审，高风险项已关闭或有豁免记录。
4. [ ] `回滚预案.md` 已确认并可执行。
5. [ ] 关键配置（插件开关、环境变量）与目标环境一致。

## 2. 发布执行
1. [ ] 锁定发布窗口并通知相关方。
2. [ ] 记录发布版本号、commit、执行人、时间。
3. [ ] 完成部署并确认健康检查通过。
4. [ ] 执行冒烟验证（至少 C1、C2、C3、C4）。

## 3. 发布后验证
1. [ ] 检查错误日志与告警面板 30 分钟无异常峰值。
2. [ ] 抽样验证中断恢复链路与委托链路。
3. [ ] 归档本次 `回归结果.json` 与 `验收报告`。
