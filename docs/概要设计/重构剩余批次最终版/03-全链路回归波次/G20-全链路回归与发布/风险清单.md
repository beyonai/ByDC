## 2. 回滚策略

1. 若发布后出现严重回归，优先执行 `git revert <发布提交>` 回滚。
2. 若需快速恢复旧路径，可从历史提交恢复关键文件：
   - `git checkout <commit> -- packages/datacloud-analysis/src/datacloud_analysis/orchestration/...`
3. 回滚后立即补充回归报告与故障复盘记录。
