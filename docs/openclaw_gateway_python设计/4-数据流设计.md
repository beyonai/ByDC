## 4.2 SDK 调用流程对比

### 原 WebSocket 流程 vs SDK 流程

| 步骤 | WebSocket 版本 | SDK 版本 |
|------|---------------|----------|
| 1. 连接 | WebSocket 握手 | 直接函数调用 |
| 2. 认证 | connect 帧 | 配置初始化 |
| 3. 发送消息 | send 帧 | `client.chat()` |
| 4. 接收响应 | agent 事件流 | 返回值/异步迭代器 |
| 5. 断开 | close 帧 | 自动资源释放 |

### 流式输出示例

```python
# SDK 流式输出 (替代 WebSocket 事件流)
async for chunk in client.chat_stream("讲一个故事"):
    print(chunk.text, end="")
    
# 或使用回调
client.on_event(lambda event: print(event))
```

## 4.2 Agent 切换流程

```
用户输入: "/model beta"
    │
    ▼
┌─────────────────┐
│ CommandRouter   │
│ cmd_model()     │
│ 解析出 switch_agent │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ SessionManager  │
│ switch_agent()  │
│                 │
│ 1. 保存当前会话状态│
│ 2. 构建新 session_key│
│    "agent:beta:main" │
│ 3. 加载或创建新会话 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 返回确认消息     │
│ "已切换到 beta"  │
│ 新 Agent 配置生效 │
└─────────────────┘
```
