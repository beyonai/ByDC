## 5.2 四层文件架构 (Four-Layer Context Architecture)

基于 OpenClaw 的上下文工程实践，工作区采用四层文件架构实现可预期的 Agent 行为：

```
workspace/
├── SOUL.md              # 身份层: 角色定位与边界（宪法）
├── IDENTITY.md          # 身份层: Agent名片（5-10行，外部系统读取）
├── USER.md              # 身份层: 用户说明书（偏好、禁止项）
├── AGENTS.md            # 操作层: 会话启动Boot Sequence
├── HEARTBEAT.md         # 操作层: 自愈机制（故障后监控）
├── TOOLS.md             # 操作层: 工具环境说明书
├── BOOTSTRAP.md         # 操作层: 冷启动初始化（仅首次）
├── MEMORY.md            # 知识层: 长期行为固件（提炼的规则）
└── memory/              # 知识层: 每日日志（原始事实）
    └── 2026-03-08.md
```

**加载顺序**（硬约束，影响覆盖优先级）:
1. **身份层** → 确定"我是谁" (SOUL → IDENTITY → USER)
2. **操作层** → 确定"怎么做" (AGENTS → TOOLS → HEARTBEAT)
3. **知识层** → 恢复"学到了什么" (MEMORY → daily logs)

**冲突解决原则**: 上层覆盖下层，SOUL.md是"宪法"，MEMORY.md是"判例"

**文件所有权**（单写多读原则 - Single Writer, Multiple Readers）:

| 文件 | 写入者 | 读取者 | 说明 |
|------|--------|--------|------|
| USER.md | 用户 | 所有Agent | 用户偏好，极少变更 |
| AGENTS.md | 用户 | 对应Agent | 每次会话启动必读 |
| SOUL.md | 用户 | 对应Agent | 角色定义，月级更新 |
| TOOLS.md | 用户 | 对应Agent | 环境变化时更新 |
| MEMORY.md | Agent自己 | Agent自己 | 私有记忆，从daily logs提炼 |
| daily logs | Agent | Agent/用户 | 任务结束后立即写入 |

**截断与预算控制**:
- 单文件最大: 20,000 字符（可配置 `bootstrap_max_chars`）
- 总预算最大: 150,000 字符（可配置 `bootstrap_total_max_chars`）
- 截断策略: 保留头部70% + 尾部20%，中间标记 `[...truncated...]`
