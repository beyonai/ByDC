### 全局 DeprecationWarning（技术债）

所有 53 个单测文件运行时均输出：

```
conftest.py:42: DeprecationWarning: There is no current event loop
    loop = asyncio.get_event_loop()
```

Python 3.10+ 中 `asyncio.get_event_loop()` 在无当前事件循环时已废弃，Python 3.12 将升级为
`DeprecationError`。属于横跨全局的技术债，需在 `conftest.py:42` 改为
`asyncio.new_event_loop()` 或使用 `pytest-asyncio` 托管事件循环。

### 新增（建议最先落地）

#### 1. 澄清流程端到端（E2E）

覆盖链路：`ambiguous → ClarificationNeededError → resume → params 回填 → dispatch 成功`

> **区分说明**：
> - **修复现有失败**：`test_query_clarification_plugin_v3.py` 中 T5_1 / T5_3 / T7_1 已覆盖
>   部分节点，修复方向见批次002 簇 E。
> - **真正缺失的新场景**：完整跨节点链路（从 `tool_dispatcher_node` 抛出
>   `ClarificationNeededError` → 写入 state → resume 后参数回填 → 第二次 dispatch 成功）
>   尚无单一测试覆盖，需新增。

#### 2. `stream_llm_call_with_retry` 调用路径覆盖

当前 `test_llm_retry.py` 全部通过，但 `llm_retry.py` 存在三处运行时缺陷（`async` 缺失、
变量覆盖、`NameError`，详见架构优化方案 P0-2），现有测试绕开了实际调用路径，形成"假绿"。

需新增：

- `async def stream_llm_call_with_retry` 可被正常 `await` 调用（验证函数可调用性）。
- `max_retries` 使用 `_DEFAULT_MAX_RETRIES`（= 3），`min_wait` 正确赋值。
- 5xx 错误 → 触发重试，等待符合指数退避。
- 429 → 在指数退避基础上追加 `Retry-After`。
- 401/403 → 立即抛出，不等待。
- 重试耗尽 → 抛出原始异常。

#### 3. LLM fallback 契约守卫

- 明确 `thinking_message_id` 必填行为；
- 主模型失败 → 备用模型接管路径；
- 主备都失败时的 checkpoint 语义。

#### 4. Schema 快照测试

固定 query/compute schema 的 `required/properties/examples`，避免字段键名反复震荡。

#### 5. Prompt 规则快照测试

固定 metrics/dimensions 的关键禁令文案，防止改文案导致协议回退。

#### 6. 可选依赖分层测试

对 `dill/by_framework` 相关测试使用显式 marker 与 skip 条件，确保"缺依赖不影响 collect"。
