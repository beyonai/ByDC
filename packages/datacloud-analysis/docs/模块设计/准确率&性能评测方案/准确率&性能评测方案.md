# 准确率 & 性能评测方案

## 一、评测目标

1. **准确率评测**：依据 `ONTOLOGY_LOAD_MODE` 分两个分支——`ontology_query` 模式下验证 `query_{code}` / `compute_{code}` 类查询；`db_query` 模式下验证 `data_query_{code}` 类查询
2. **性能评测**：两项硬性指标
   - **首字节时延（TTFB）≤ 1000 ms**：从请求到达到首个 ANSWER_DELTA 文本 chunk 发出的时间
   - **流式输出速度 ≥ 20 字符/秒**：ANSWER_DELTA 文本 chunk 的总字符数 / 流式持续时间
3. **可追溯性**：每个问答具备唯一 ID，全链路日志、性能、误差可查
4. **误差溯源**：评测失败时，自动定位具体环节/函数/原因

## 三、评测架构

**核心原则：通过 DataCloudWorker 调用，复用完整生产路径**

现有 Agent 系统的实际调用链：

```
GatewayWorker._handle_message()
  └─► DataCloudWorker.process_command()   (worker.py)
        └─► _build_graph() → create_agent()
              ├─ prompts_dict   ← plugin_registry（数字员工配置）
              ├─ tools_dict     ← InitDataCloudDigitalEmployeePlugin
              ├─ mounted_objects← 本体对象列表
              └─ loader         ← OntologyLoader（真实数据库连接）
```

评测方案**通过 DataCloudWorker.process_command() 直接驱动**，并在子类中注入 EvaluationCallback：

```
┌─────────────────────────────────────────────────────────────────┐
│                      Evaluation Runner                           │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────┐   │
│  │ Case Loader  │  │ Judge        │  │ Report Generator    │   │
│  └──────────────┘  └──────────────┘  └──────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
                    │  AskAgentCommand + ComplexAgentContext
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│      EvalDataCloudWorker.process_command()  (worker 子类)        │
│  · 从 plugin_registry 读取数字员工配置                            │
│  · 构建/缓存 langgraph 图（含真实工具 + OntologyLoader）          │
```

### 评测依赖

| 依赖 | 说明 |
|-----|------|
| Redis | 会话历史（`session_manager.history`）+ langgraph checkpoint |
| plugin_registry | 数字员工配置（prompts / tools / mounted_objects）|
| `ONTOLOGY_LOAD_MODE` | 决定工具命名和加载路径（见下节）|
| OntologyLoader | `ontology_query` 模式下 schema 查询所需；`db_query` 模式下静态建工具时也需要 |
| ComplexAgentContext | 替代 by_framework.AgentContext，支持 paradigm 交互 |

### ONTOLOGY_LOAD_MODE 两种模式对比

`ONTOLOGY_LOAD_MODE` 决定了 `InitDataCloudDigitalEmployeePlugin` 在 `start_heartbeat()` 时如何为数字员工构建工具，进而影响 Agent 可调用的工具名称和评测用例的期望值。

| 维度 | `ontology_query`（默认） | `db_query` |
|------|------------------------|------------|
| 工具命名 | `query_{code}`、`compute_{code}` | `data_query_{code}` |
| 工具来源 | 运行时由 `OntologyToolLoader` 动态生成 | 启动时由 `_build_db_query_tools` 静态构建 |
| `mounted_objects` | 有值（OBJECT/VIEW resource code 列表）| 空列表 |
| `loader`（AgentConfig.extra） | `OntologyLoader` 实例 | `None` |
| 评测 `expected_tools` 键名 | `query_ads_manage_grid_analysis` 等 | `data_query_ads_manage_grid_analysis` 等 |
| 适用场景 | 有本体知识库、需要语义推理 | 纯 DB 直查、无本体依赖 |

**两种模式共存时的评测策略**：用例 `expected_tools` 按模式拆分存放，评测运行时读取当前 `ONTOLOGY_LOAD_MODE` 选择对应分支：

```python
# 用例格式（两个模式的期望工具分开声明）
{
    "case_id": "TC_001",
    "expected_intent": "query_ads_manage_grid_analysis",
    "expected_tools": {
        "ontology_query": ["query_ads_manage_grid_analysis"],
        "db_query":       ["data_query_ads_manage_grid_analysis"]
    },
    ...
}

# Judge 在评判时按当前模式取对应期望值
import os
_mode = os.environ.get("ONTOLOGY_LOAD_MODE", "ontology_query").strip().lower()
actual_expected_tools = case["expected_tools"].get(_mode, [])
```

多轮对话状态管理：`thread_id = f"{agent_key}:{session_id}"`，同一会话复用相同 `session_id` 即可自动恢复上下文（langgraph checkpoint + Redis 历史双保险）。

### 评测程序与 DataCloudWorker 的集成

```python
from langgraph.callbacks import CallbackHandler
import uuid
import time

class EvaluationCallback(CallbackHandler):
    """评测专用回调：捕获每个节点的输入输出、耗时、错误"""

    def __init__(self):
        self.spans = []
        self.trace_id = str(uuid.uuid4())
        self._node_stack = []  # 嵌套调用栈

    def on_chain_start(self, run_id, name, input, **kwargs):
        self.spans.append({
            "span_id": run_id,
            "trace_id": self.trace_id,
            "node": name,
            "event": "start",
            "start_time": time.time(),
            "input": str(input)[:200] if input else None
        })
        self._node_stack.append(run_id)

    def on_chain_end(self, run_id, name, output, **kwargs):
        if self._node_stack and self._node_stack[-1] == run_id:
            self._node_stack.pop()

        for span in reversed(self.spans):
            if span["span_id"] == run_id and span["event"] == "start":
                span["event"] = "end"
                span["end_time"] = time.time()
                span["duration_ms"] = (span["end_time"] - span["start_time"]) * 1000
                span["output"] = str(output)[:500] if output else None
                break

    def on_tool_start(self, run_id, name, input, **kwargs):
        """工具调用开始 (SQL 执行等)"""
        self.spans.append({
            "span_id": f"{run_id}_tool",
            "trace_id": self.trace_id,
            "node": name,
            "layer": "tool",
            "event": "start",
            "start_time": time.time()
        })

    def on_tool_end(self, run_id, name, output, **kwargs):
        """工具调用结束"""
        for span in reversed(self.spans):
            if span["span_id"] == f"{run_id}_tool":
                span["event"] = "end"
                span["end_time"] = time.time()
                span["duration_ms"] = (span["end_time"] - span["start_time"]) * 1000
                break
```

### 评测程序调用 Agent 的方式

```python
import asyncio
import uuid
from by_framework.core.protocol.commands import AskAgentCommand, CommandHeader
from byclaw_data.custom_agent_context import ComplexAgentContext
from byclaw_data.worker import DataCloudWorker
from .callbacks.evaluation_callback import EvaluationCallback


async def run_evaluation(
    case: TestCase,
    worker: "EvalDataCloudWorker",   # 已初始化：redis 已连, plugin_registry 已加载
    agent_id: str,
    session_id: str | None = None,   # 多轮复用同一 session_id，单轮传 None 自动生成
    workspace_dir: str = "/tmp/eval",
):
    session_id = session_id or str(uuid.uuid4())
    message_id = str(uuid.uuid4())
    trace_id   = str(uuid.uuid4())

    # 构造 gateway 协议命令（模拟前端入站 AskAgentCommand）
    command = AskAgentCommand(
        content=case.query,
        header=CommandHeader(
            session_id=session_id,
            message_id=message_id,
            trace_id=trace_id,
            target_agent_type=agent_id,
            tenant_id="eval",
            metadata={"agent_id": agent_id},
        )
    )

    # 构造 ComplexAgentContext（含 Redis + plugin_registry，与生产一致）
    context = ComplexAgentContext(
        session_id=session_id,
        trace_id=trace_id,
        redis_client=worker.redis,              # 生产同款 Redis
        current_agent_id=agent_id,
        parent_message_id=message_id,
        current_command=command,
        cancel_event=asyncio.Event(),
        cancel_reason="",
        plugin_registry=worker.plugin_registry, # 含数字员工配置
        tenant_id="eval",
        workspace_dir=workspace_dir,
        agent_configs=worker.plugin_registry.agent_configs,
        storage=worker.storage,
    )

    # 调用生产路径（内部完成：配置解析→建图→thread_id→stream_graph）
    # EvaluationCallback 由 EvalDataCloudWorker._stream_graph() 自动注入
    result = await worker.process_command(command, context)

    return {
        "trace_id": trace_id,
        "session_id": session_id,
        "thread_id": worker._build_thread_id(
            session_id=session_id, agent_key=agent_id
        ),
        "result": result,
        "spans": worker.last_eval_callback.spans if worker.last_eval_callback else [],
    }
```

### 优势

1. **完全复用生产路径** - `process_command()` 内部完成配置解析、建图、工具加载
2. **多轮对话自动延续** - 同一 `session_id` 下 `thread_id` 不变，checkpoint 自动恢复
3. **真实工具和数据库** - `OntologyLoader` 连接真实数据源，不需要 mock
4. **回调注入无侵入** - 通过 `EvalDataCloudWorker` 子类覆盖 `_stream_graph()`，不修改生产代码

### 4.1 评测用例 (TestCase)

```python
@dataclass
class TestCase:
    case_id: str                    # 唯一标识: "TC_001"
    category: str                   # "single_turn" | "multi_turn" | "chain_query"
    turn_index: int                  # 多轮中的轮次
    query: str                       # 用户问题
    expected_intent: str | Dict      # 期望的意图（见下方说明）
    # 可为字符串或按模式分支的字典：
    #   字符串格式（仅供文档可读性，Judge 会用 expected_tools[mode][0] 自动覆盖）
    #   字典格式（显式声明，优先级最高）：
    #     {"ontology_query": "compute_ads_manage_grid_analysis",
    #      "db_query":       "data_query_ads_manage_grid_analysis"}
    # Judge._resolve_expected_intent() 解析规则：
    #   1. 若为 dict → 取 dict[当前 mode]
    #   2. 否则 → 取 expected_tools[当前 mode][0]（自动推导，与工具名保持一致）
    expected_tools: List[str] | Dict  # 期望调用的工具（dict 格式按 mode 分支）
    expected_sql_pattern: str        # SQL 模式正则 (用于校验)
    expected_fields: List[str]      # 期望返回的字段
    expected_conditions: Dict       # 期望的查询条件
    difficulty: str                  # "easy" | "medium" | "hard"
    tags: List[str]                 # ["query", "compute", "filter"]
    expected_tool_args: Dict | None = None
    # 期望的工具调用参数（按 ONTOLOGY_LOAD_MODE 分支）。格式：
    #   {"ontology_query": {<tool_params>}, "db_query": {<tool_params>}}
    # tool_params 依工具类型不同：
    #   query_{code}   → {select, filters, filter_relation, order_by, limit, offset}
    #   compute_{code} → {dimensions, metrics, filters, having, order_by, limit}
    #   data_query_{code}（db_query 模式）→ 参数 schema 与目标模式平台相关
    # Judge 只校验 expected_tool_args 中声明的键（partial match）。
    # 若当前模式对应的键不存在（如 db_query 键缺失），则跳过本维度校验。
    expected_error: Dict | None = None
    # 若用例预期工具调用返回错误，则设置此字段。两种格式：
    #
    # 1. 扁平格式（两个模式共用）：
    #      {"error_code": "VIRTUAL_ACTION_ERR_...", "message_pattern": "..."}
    #
    # 2. 模式键控格式（不同模式下错误行为不同时使用）：
    #      {"ontology_query": {"error_code": "...", "message_pattern": "..."},
    #       "db_query": null}
    #    某模式值为 null 表示该模式下不校验错误（跳过）。
    #
    # VIRTUAL_ACTION_ERR_* 错误码由 ontology_query 模式的工具层生成；
    # db_query 模式工具无该校验层，对应用例应将 "db_query" 设为 null。
    expected_tool_chain: List[Dict] | None = None
    # 链式查询用例专用：描述单次用户提问中 Agent 需按顺序发起的多次工具调用。
    # 与 expected_tool_args 的区别：
    #   expected_tool_args  → 单工具调用，校验参数内容
    #   expected_tool_chain → 多工具调用，校验调用顺序 + 结果传递
    # 格式（list 按 step 顺序排列）：
    # [
    #   {
    #     "step": 1,
    #     "tool": {"ontology_query": "compute_xxx", "db_query": "data_query_xxx"},
    #     "purpose": "...",          # 该步骤目的（文档说明用）
    #     "key_output_field": "...", # 该步骤结果中传递给下一步的字段（中文名）
    #     "filter_from_step": null   # 无前置依赖
    #   },
    #   {
    #     "step": 2,
    #     "tool": {"ontology_query": "query_yyy", "db_query": "data_query_yyy"},
    #     "purpose": "...",
    #     "key_output_field": "...",
    #     "filter_from_step": 1,     # 依赖 step 1 的输出
    #     "filter_field": "..."      # step 1 中哪个字段的值被用作本步骤的 filter value
    #   },
    #   ...
    # ]
    # Judge 校验：① 调用顺序正确；② 每步 tool 名称匹配；③ step N 的 filters 中包含 step N-1 的 key_output_field 值
```

### 4.2 执行轨迹 (Trace)

```python
@dataclass
class PerformanceMetrics:
    """单次调用的性能指标，由 EvalComplexAgentContext 在 emit_chunk 时采集。"""
    request_start_time: float        # process_command 开始的 monotonic 时间
    first_chunk_time: float | None   # 首个 ANSWER_DELTA 文本 chunk 的 monotonic 时间
    last_chunk_time: float | None    # 最后一个 ANSWER_DELTA 文本 chunk 的 monotonic 时间
    total_chars: int                 # 所有 ANSWER_DELTA 文本 chunk 的字符总数

    @property
    def ttfb_ms(self) -> float | None:
        """首字节时延（ms）。首 chunk 未收到时为 None。"""
        if self.first_chunk_time is None:
            return None
        return (self.first_chunk_time - self.request_start_time) * 1000

    @property
    def stream_duration_ms(self) -> float | None:
        """流式持续时长（ms）：首 chunk → 末 chunk。"""
        if self.first_chunk_time is None or self.last_chunk_time is None:
            return None
        return (self.last_chunk_time - self.first_chunk_time) * 1000

    @property
    def output_speed_chars_per_sec(self) -> float | None:
        """输出速度（字符/秒）。持续时间为 0 或无流式输出时为 None。"""
        dur = self.stream_duration_ms
        if dur is None or dur <= 0:
            return None
        return self.total_chars / (dur / 1000)

    # 阈值判断
    TTFB_THRESHOLD_MS: float = 1000       # 首字节时延上限
    OUTPUT_SPEED_MIN: float = 20          # 最低输出速度（字符/秒）

    @property
    def ttfb_pass(self) -> bool:
        t = self.ttfb_ms
        return t is not None and t <= self.TTFB_THRESHOLD_MS

    @property
    def output_speed_pass(self) -> bool:
        s = self.output_speed_chars_per_sec
        return s is not None and s >= self.OUTPUT_SPEED_MIN

@dataclass
class Trace:
    trace_id: str
    case_id: str
    query: str
    start_time: float
    end_time: float
    duration_ms: float
    status: str                     # "success" | "partial" | "failed"
    spans: List[Span]
    final_response: str
    expected_response: str
    accuracy: float                 # 0.0 ~ 1.0
    perf: PerformanceMetrics | None = None
    tool_call_sequence: List[Dict] = field(default_factory=list)
    # 有序工具调用列表（含重复），每项格式：{"tool": str, "args": dict}
    # 用于链式查询校验；单工具用例时长度为 1。
    # 由 EvaluationRunner._extract_tool_call_sequence() 从 messages 中提取。

@dataclass
class Span:
    span_id: str
    trace_id: str
    layer: str                      # "agent" | "tool" | "llm" | "db"
    name: str                       # "IntentClassifier.classify"
    start_time: float
    end_time: float
    duration_ms: float
    input_data: Dict
    output_data: Dict
    error: Optional[str]

@dataclass
class ErrorInfo:
    span_id: str
    error_type: str                 # "sql_error" | "intent_error" | "field_missing"
    error_message: str
    expected: Any
    actual: Any
    severity: str                   # "critical" | "major" | "minor"
```

#### TC_Q_001: 派生指标 formula 展开（SELECT / WHERE / ORDER BY 均展开公式）


### 5.4 compute_ontology 参数规则验收用例

> 覆盖 §3.2.3（compute_{code} 方法）的分组统计、having、agg 约束、错误场景。
> `compute_` 方法无 `offset` 参数；默认排序方向为 `desc`（与 `query_` 的 `asc` 相反）。

#### TC_C_001: 多维度分组统计 + count_all 内建指标

```json
{
    "case_id": "TC_C_001",
    "category": "single_turn",
    "turn_index": 1,
    "query": "按月和管理网格统计2025年下半年各月的企业数量和总税收",

        "ontology_query": ["compute_ads_manage_grid_analysis"],
        "db_query":       ["data_query_ads_manage_grid_analysis"]
    },
    "expected_sql_pattern": "GROUP BY.*DATE_FORMAT.*period.*COUNT.*SUM.*total_tax",
    "expected_fields": ["manage_grid_name", "total_tax"],
    "expected_conditions": {},
    "expected_tool_args": {
        "ontology_query": {
            "dimensions": [
                {"field": "账期",       "group_op": "month"},
                {"field": "管理网格名称", "group_op": "self"}
            ],
            "metrics": [
                {"agg": "count_all",                     "as": "ent_count"},
                {"field": "总税收", "agg": "sum",         "as": "total_tax"}
            ],
            "filters": [
                {"field": "账期", "op": "between", "value": ["2025-07", "2025-12"]}
```

> 验证要点：
> 1. `"账期"` 维度使用 `group_op=month`，SQL 展开为 `DATE_FORMAT(period_month,'%Y-%m')`
> 2. `count_all` 内建指标不需要 `field`，直接生成 `COUNT(*) AS "ent_count"`
> 3. `order_by` 引用 `metrics.as` 别名 `total_tax`，默认方向 `desc`

### 5.6 链式查询用例（单问题多步骤）

> 本节用例对应用户在**一次提问**中隐含的 A → B → C 三级子查询。与多轮（5.2）的区别：
> - **多轮**：用户发送多条消息，每条消息独立触发 Agent
> - **链式**：用户只发一条消息，Agent 自主规划并顺序发起多次工具调用，将前步结果作为后步的过滤条件
>
> 评测关注三点：
> 1. Agent 能否正确识别三个子任务并按顺序调用对应工具
> 2. 每步工具调用是否将前步结果正确传递（作为 filters/dimensions 中的字段值）
> 3. 最终回答是否综合三步结果给出完整结论
>
> `expected_tool_chain` 中的 `filter_field` 填**中文字段名**（LLM 视角），Judge 检查实际 filters 中是否包含该字段名。

```python
# callbacks/evaluation_callback.py
from langgraph.callbacks import CallbackHandler
from dataclasses import dataclass, field
from typing import Any, Optional
import uuid
import time

@dataclass
class EvalSpan:
    span_id: str
    trace_id: str
    node: str                    # intend / execution / respond / tool
    layer: str                  # "agent" | "llm" | "tool" | "db"
    event: str                  # "start" | "end"
    start_time: float = 0
    end_time: float = 0
    duration_ms: float = 0
    input_data: Any = None
    output_data: Any = None
    error: Optional[str] = None

class EvaluationCallback(CallbackHandler):
    """
    评测专用回调，注入到 langgraph Agent 的 config["callbacks"] 中。
    复用 langgraph 内置的 CallbackHandler 体系，无需 Hook Agent 代码。
    """

    def __init__(self):
        self.trace_id = str(uuid.uuid4())
        self.spans: list[EvalSpan] = []
        self._stack: list[str] = []  # 嵌套调用栈

    # === langgraph 回调接口 ===

    def on_chain_start(self, run_id: str, name: str, input: Any, **kwargs):
        """节点开始 (intend / execution / respond)"""
        span = EvalSpan(
            span_id=run_id,
            trace_id=self.trace_id,
            node=name,
            layer="agent",
            event="start",
            start_time=time.time(),
            input_data=input
        )
        self.spans.append(span)
        self._stack.append(run_id)

    def on_chain_end(self, run_id: str, name: str, output: Any, **kwargs):
        """节点结束"""
        if self._stack and self._stack[-1] == run_id:
            self._stack.pop()
        for span in reversed(self.spans):
            if span.span_id == run_id and span.event == "start":
                span.event = "end"
                span.end_time = time.time()
                span.duration_ms = (span.end_time - span.start_time) * 1000
                span.output_data = output
                break

    def on_chain_error(self, run_id: str, name: str, error: Exception, **kwargs):
        """节点异常"""
        for span in reversed(self.spans):
            if span.span_id == run_id and span.event == "start":
                span.event = "error"
                span.end_time = time.time()
                span.duration_ms = (span.end_time - span.start_time) * 1000
                span.error = str(error)
                break

    def on_chat_model_start(self, run_id: str, name: str, input: Any, **kwargs):
        """LLM 调用开始"""
        span = EvalSpan(
            span_id=f"{run_id}_llm",
            trace_id=self.trace_id,
            node=f"llm:{name}",
            layer="llm",
            event="start",
            start_time=time.time(),
            input_data=str(input)[:300]
        )
        self.spans.append(span)

    def on_chat_model_end(self, run_id: str, name: str, output: Any, **kwargs):
        """LLM 调用结束"""
        for span in reversed(self.spans):
            if span.span_id == f"{run_id}_llm":
                span.event = "end"
                span.end_time = time.time()
                span.duration_ms = (span.end_time - span.start_time) * 1000
                span.output_data = str(output)[:300]
                break

    def on_tool_start(self, run_id: str, name: str, input: Any, **kwargs):
        """工具调用开始 (SQL 执行等)"""
        span = EvalSpan(
            span_id=f"{run_id}_tool",
            trace_id=self.trace_id,
            node=f"tool:{name}",
            layer="tool",
            event="start",
            start_time=time.time(),
            input_data=str(input)[:200]
        )
        self.spans.append(span)

    def on_tool_end(self, run_id: str, name: str, output: Any, **kwargs):
        """工具调用结束"""
        for span in reversed(self.spans):
            if span.span_id == f"{run_id}_tool":
                span.event = "end"
                span.end_time = time.time()
                span.duration_ms = (span.end_time - span.start_time) * 1000
                span.output_data = str(output)[:500]
                break

    def on_tool_error(self, run_id: str, name: str, error: Exception, **kwargs):
        """工具调用错误"""
        for span in reversed(self.spans):
            if span.span_id == f"{run_id}_tool":
                span.event = "error"
                span.end_time = time.time()
                span.duration_ms = (span.end_time - span.start_time) * 1000
                span.error = str(error)
                break

    def get_trace_summary(self) -> dict:
        """生成追踪摘要"""
        agent_spans = [s for s in self.spans if s.layer == "agent"]
        total_ms = sum(s.duration_ms for s in self.spans)
        return {
            "trace_id": self.trace_id,
            "total_duration_ms": total_ms,
            "span_count": len(self.spans),
            "node_durations": {
                s.node: s.duration_ms for s in self.spans if s.event == "end"
            },
            "spans": [
                {
                    "node": s.node,
                    "layer": s.layer,
                    "duration_ms": s.duration_ms,
                    "error": s.error
                }
                for s in self.spans if s.event in ("end", "error")
            ]
        }
```

### 6.3 EvalComplexAgentContext（采集 TTFB / 输出速度）

```python
# evaluation/eval_context.py
"""
EvalComplexAgentContext: 在 ComplexAgentContext 基础上拦截 emit_chunk，
采集首字节时延（TTFB）和流式输出速度所需的原始时间戳。
"""
import time
from typing import Any

from by_framework.core.protocol.event_type import EventType
from by_framework.core.protocol.content_type import SseMessageType
from byclaw_data.custom_agent_context import ComplexAgentContext


class EvalComplexAgentContext(ComplexAgentContext):
    """
    评测专用 AgentContext。
    覆盖 emit_chunk()，对 ANSWER_DELTA + text 类型的 chunk 进行时间戳和字符计数采集：
      - first_chunk_time : 首个文本 chunk 的 monotonic 时间（用于计算 TTFB）
      - last_chunk_time  : 末个文本 chunk 的 monotonic 时间（用于计算流式持续时间）
      - total_chars      : 所有文本 chunk 的字符总数
    request_start_time 由 runner 在调用 process_command() 前设置。
    """

    def __init__(self, *args: Any, request_start_time: float | None = None, **kwargs: Any):
        super().__init__(*args, **kwargs)
        self.request_start_time: float = request_start_time or time.monotonic()
        self.first_chunk_time: float | None = None
        self.last_chunk_time: float | None = None
        self.total_chars: int = 0

    async def emit_chunk(self, chunk: Any, *, event_type: str | None = None,
                         content_type: str | None = None, **kwargs: Any) -> None:
        """拦截 ANSWER_DELTA + text 类 chunk，采集时间戳和字符数。"""
        is_answer_delta = (event_type == EventType.ANSWER_DELTA.value)
        is_text_chunk = (content_type is None or content_type == SseMessageType.text.value)

        if is_answer_delta and is_text_chunk:
            content: str = getattr(chunk, "content", "") or ""
            if isinstance(content, str) and content:
                now = time.monotonic()
                if self.first_chunk_time is None:
                    self.first_chunk_time = now
                self.last_chunk_time = now
                self.total_chars += len(content)

        await super().emit_chunk(chunk, event_type=event_type,
                                 content_type=content_type, **kwargs)

    @property
    def ttfb_ms(self) -> float | None:
        if self.first_chunk_time is None:
            return None
        return (self.first_chunk_time - self.request_start_time) * 1000

    @property
    def stream_duration_ms(self) -> float | None:
        if self.first_chunk_time is None or self.last_chunk_time is None:
            return None
        return (self.last_chunk_time - self.first_chunk_time) * 1000

    @property
    def output_speed_chars_per_sec(self) -> float | None:
        dur = self.stream_duration_ms
        if dur is None or dur <= 0:
            return None
        return self.total_chars / (dur / 1000)
```

```python
import asyncio
from typing import Any

class EvalDataCloudWorker(DataCloudWorker):
```

#### EvaluationRunner（使用 EvalDataCloudWorker）

```python
# evaluation/runner.py
import asyncio
import time
import uuid
from dataclasses import dataclass
from typing import Any, Dict, List

from by_framework.core.protocol.commands import AskAgentCommand, CommandHeader

from .callbacks.evaluation_callback import EvaluationCallback
from .eval_context import EvalComplexAgentContext
from .eval_worker import EvalDataCloudWorker
from .judge import Judge
from .cases.loader import CaseLoader


@dataclass
class EvaluationConfig:
    worker: EvalDataCloudWorker     # 已初始化的 EvalDataCloudWorker
    agent_id: str                   # 数字员工 ID（BYAI_DIG_EMPLOYEE_{id}.json）
    cases_file: str
    output_dir: str
    workspace_dir: str = "/tmp/eval"
    performance_threshold_ms: float = 5000


class EvaluationRunner:
    """
    评测运行器：通过 EvalDataCloudWorker.process_command() 调用，
    完整复用生产路径（数字员工配置 / 真实工具 / Redis checkpoint）。
    """

    def __init__(self, config: EvaluationConfig):
        self.config = config
        self.judge = Judge()
        self.case_loader = CaseLoader()

    # ------------------------------------------------------------------
    # 内部工具
    # ------------------------------------------------------------------

    def _make_command(self, query: str, session_id: str) -> AskAgentCommand:
        """构造 AskAgentCommand，模拟 gateway 入站消息。"""
        return AskAgentCommand(
            content=query,
            header=CommandHeader(
                session_id=session_id,
                message_id=str(uuid.uuid4()),
                trace_id=str(uuid.uuid4()),
                target_agent_type=self.config.agent_id,
                tenant_id="eval",
                metadata={"agent_id": self.config.agent_id},
            ),
        )

    def _make_context(
        self, command: AskAgentCommand, session_id: str,
        request_start_time: float | None = None,
    ) -> "EvalComplexAgentContext":
        """构造 EvalComplexAgentContext，含 Redis + plugin_registry，与生产一致。"""
        worker = self.config.worker
        return EvalComplexAgentContext(
            session_id=session_id,
            trace_id=command.header.trace_id,
            redis_client=worker.redis,
            current_agent_id=self.config.agent_id,
            parent_message_id=command.header.message_id,
            current_command=command,
            cancel_event=asyncio.Event(),
            cancel_reason="",
            plugin_registry=worker.plugin_registry,
            tenant_id="eval",
            workspace_dir=self.config.workspace_dir,
            agent_configs=worker.plugin_registry.agent_configs,
            storage=worker.storage,
            request_start_time=request_start_time,
        )

    async def _invoke_turn(self, query: str, session_id: str) -> Dict[str, Any]:
        """执行一轮对话并返回 trace 摘要（含 TTFB / 输出速度）。"""
        worker = self.config.worker
        command = self._make_command(query, session_id)
        t_start = time.monotonic()
        context = self._make_context(command, session_id, request_start_time=t_start)

        await worker.process_command(command, context)

        # 从 EvalComplexAgentContext 读取性能指标
        ttfb_ms = context.ttfb_ms
        output_speed = context.output_speed_chars_per_sec

        callback: EvaluationCallback | None = worker.last_eval_callback
        spans = callback.spans if callback else []
        summary = callback.get_trace_summary() if callback else {}

        # 执行完毕后，通过 aget_state 读取最终图状态（含 messages）
        thread_id = worker._build_thread_id(
            session_id=session_id, agent_key=self.config.agent_id
        )
        graph_state = None
        for graph in worker.graphs.values():
            try:
                snapshot = await graph.aget_state(
                    {"configurable": {"thread_id": thread_id}}
                )
                graph_state = snapshot
                break
            except Exception:
                continue

        messages = []
        if graph_state and hasattr(graph_state, "values"):
            messages = graph_state.values.get("messages", [])

        return {
            "trace_id": command.header.trace_id,
            "session_id": session_id,
            "thread_id": thread_id,
            "messages": messages,
            "intent": self._extract_intent(messages),
            "tools_used": self._extract_tools(messages),
            "tool_call_sequence": self._extract_tool_call_sequence(messages),
            "generated_sql": self._extract_sql(messages),
            "returned_fields": self._extract_fields(messages),
            "tool_call_args": self._extract_tool_call_args(messages),
            "actual_error": self._extract_actual_error(messages),
            "total_duration_ms": summary.get("total_duration_ms", 0),
            "spans": spans,
            "ttfb_ms": ttfb_ms,
            "output_speed_chars_per_sec": output_speed,
            "ttfb_pass": ttfb_ms is not None and ttfb_ms <= 1000,
            "output_speed_pass": output_speed is not None and output_speed >= 20,
        }

    # ------------------------------------------------------------------
    # 单轮 / 多轮评测
    # ------------------------------------------------------------------

    async def run_single_turn(self, case: Dict) -> Dict:
        """执行单轮评测：每个 case 独立 session_id。"""
        session_id = str(uuid.uuid4())
        trace = await self._invoke_turn(case["query"], session_id)

        judge_result = self.judge.judge(case, trace)
        return {
            "case_id": case["case_id"],
            "trace_id": trace["trace_id"],
            "status": judge_result.status,
            "accuracy_score": judge_result.accuracy_score,
            "performance_score": judge_result.performance_score,
            "ttfb_ms": trace.get("ttfb_ms"),
            "output_speed_chars_per_sec": trace.get("output_speed_chars_per_sec"),
            "ttfb_pass": judge_result.ttfb_pass,
            "output_speed_pass": judge_result.output_speed_pass,
            "errors": judge_result.errors,
            "spans": trace["spans"],
        }

    async def run_multi_turn(self, case: Dict) -> Dict:
        """
        执行多轮评测。
        同一 case 内所有轮次共享 session_id：
          thread_id = f"{agent_id}:{session_id}"
        worker 通过 langgraph checkpoint（Redis）自动延续上下文。
        """
        session_id = str(uuid.uuid4())   # 整个多轮会话的唯一 session
        turn_results = []

        for turn in case["turns"]:
            trace = await self._invoke_turn(turn["query"], session_id)
            judge_result = self.judge.judge(turn, trace)
            turn_results.append({
                "turn_index": turn["turn_index"],
                "trace_id": trace["trace_id"],
                "status": judge_result.status,
                "accuracy_score": judge_result.accuracy_score,
                "ttfb_ms": trace.get("ttfb_ms"),
                "output_speed_chars_per_sec": trace.get("output_speed_chars_per_sec"),
                "ttfb_pass": judge_result.ttfb_pass,
                "output_speed_pass": judge_result.output_speed_pass,
                "errors": judge_result.errors,
            })

        avg_accuracy = (
            sum(r["accuracy_score"] for r in turn_results) / len(turn_results)
            if turn_results else 0.0
        )
        ttfb_pass_all = all(r.get("ttfb_pass", False) for r in turn_results)
        output_speed_pass_all = all(r.get("output_speed_pass", False) for r in turn_results)
        return {
            "case_id": case["case_id"],
            "session_id": session_id,
            "turn_results": turn_results,
            "overall_accuracy": avg_accuracy,
            "all_passed": all(r["status"] == "pass" for r in turn_results),
            "ttfb_pass_all": ttfb_pass_all,
            "output_speed_pass_all": output_speed_pass_all,
        }

    # ------------------------------------------------------------------
    # 辅助方法：从 messages 中提取关键信息
    # ------------------------------------------------------------------

    def _extract_tool_call_sequence(self, messages: List) -> List[Dict[str, Any]]:
        """
        提取所有工具调用，保持顺序（允许重复，链式查询会多次调用不同工具）。
        每项格式：{"tool": str, "args": dict}
        """
        sequence = []
        for msg in messages:
            calls = getattr(msg, "tool_calls", None)
            if calls:
                for call in calls:
                    sequence.append({
                        "tool": call.get("name", ""),
                        "args": call.get("args", {}),
                    })
        return sequence

    def _extract_tool_call_args(self, messages: List) -> Dict[str, Any]:
        """从 AIMessage.tool_calls 中提取第一次工具调用的 name + args。"""
        for msg in messages:
            calls = getattr(msg, "tool_calls", None)
            if calls:
                first = calls[0]
                return {"tool": first.get("name", ""), "args": first.get("args", {})}
        return {}

    def _extract_actual_error(self, messages: List) -> Dict[str, Any]:
        """
        从 ToolMessage 的 content 中提取工具调用返回的错误信息。
        格式：{"error_code": "VIRTUAL_ACTION_ERR_...", "message": "..."}
        """
        import json as _json
        for msg in reversed(messages):
            content = str(getattr(msg, "content", ""))
            if "error_code" in content or "VIRTUAL_ACTION_ERR" in content:
                try:
                    data = _json.loads(content)
                    return {
                        "error_code": data.get("error_code", ""),
                        "message":    data.get("message", ""),
                    }
                except Exception:
                    pass
        return {}

    def _extract_intent(self, messages: List) -> str:
        """
        从消息列表中推断 intent：取第一次工具调用的名称。

        不依赖消息文本中的 "intent:" 关键词（LLM 输出格式不稳定），
        而是直接读取 AIMessage.tool_calls[0]["name"]，与 ONTOLOGY_LOAD_MODE
        无关——ontology_query 模式返回 query_*/compute_*，
        db_query 模式返回 data_query_*。
        """
        for msg in messages:
            calls = getattr(msg, "tool_calls", None)
            if calls:
                return calls[0].get("name", "")
        return ""

    def _extract_tools(self, messages: List) -> List[str]:
        """
        提取工具名列表（去重但保持首次出现顺序）。
        用于 tools_correct 集合比较；链式查询的顺序校验使用 tool_call_sequence。
        """
        seen: set = set()
        tools: List[str] = []
        for msg in messages:
            if hasattr(msg, "name") and msg.name and msg.name not in seen:
                seen.add(msg.name)
                tools.append(msg.name)
        return tools

    def _extract_sql(self, messages: List) -> str:
        import re
        for msg in reversed(messages):
            content = str(getattr(msg, "content", ""))
            if "SELECT" in content.upper() and "FROM" in content.upper():
                match = re.search(r"(SELECT\s+.*?;?)", content, re.IGNORECASE | re.DOTALL)
                if match:
                    return match.group(1).strip()
        return ""

    def _extract_fields(self, messages: List) -> List[str]:
        return []
```

```python
# judge.py
import os
import re
from dataclasses import dataclass
from typing import Dict, List, Any, Optional

# 读取当前加载模式，用于从用例的 expected_tools 中取对应分支
_ONTOLOGY_LOAD_MODE = os.environ.get("ONTOLOGY_LOAD_MODE", "ontology_query").strip().lower()


def resolve_expected_tools(case: Dict) -> list:
    """
    从用例的 expected_tools 中，依据当前 ONTOLOGY_LOAD_MODE 取出期望工具列表。

    支持新格式（dict）和旧格式（list，向后兼容）：
      新格式: {"ontology_query": [...], "db_query": [...]}
      旧格式: [...]  → 直接返回
    """
    raw = case.get("expected_tools", [])
    if isinstance(raw, dict):
        return raw.get(_ONTOLOGY_LOAD_MODE, [])
    return raw   # 旧格式兼容


def resolve_expected_error(case: Dict) -> Dict | None:
    """
    从用例的 expected_error 中，依据当前 ONTOLOGY_LOAD_MODE 取出期望错误描述。

    支持两种格式：
      扁平格式: {"error_code": "...", "message_pattern": "..."}  → 两模式共用
      模式键控: {"ontology_query": {...}, "db_query": null}
        → 当前模式值为 None 时表示该模式下跳过错误校验

    返回 None 表示跳过错误校验（当前模式不适用）。
    """
    raw = case.get("expected_error")
    if raw is None:
        return None
    if isinstance(raw, dict) and \
       ("ontology_query" in raw or "db_query" in raw):
        # 模式键控格式：取当前模式对应值（可能是 None）
        return raw.get(_ONTOLOGY_LOAD_MODE)
    # 扁平格式：两模式共用
    return raw


@dataclass
class JudgeResult:
    case_id: str
    trace_id: str
    status: str  # "pass", "fail", "partial"
    accuracy_score: float  # 0.0 ~ 1.0
    performance_score: float  # 0.0 ~ 1.0（基于 TTFB / 输出速度各 50%）
    intent_correct: bool
    tools_correct: bool
    sql_correct: bool
    fields_correct: bool
    ttfb_pass: bool          # TTFB ≤ 1000 ms
    output_speed_pass: bool  # 输出速度 ≥ 20 字符/秒
    tool_args_correct: bool = True   # 工具调用参数符合 expected_tool_args（仅当用例声明时有效）
    error_correct: bool = True       # 错误返回符合 expected_error（仅当用例声明时有效）
    chain_correct: bool = True       # 链式调用顺序及结果传递正确（仅 expected_tool_chain 用例有效）
    errors: List[Dict]

class Judge:
    def __init__(self):
        self.intent_weight = 0.2
        self.tools_weight = 0.2
        self.sql_weight = 0.3
        self.fields_weight = 0.2
        # 性能判断阈值（硬编码，与 PerformanceMetrics 保持一致）
        self.ttfb_threshold_ms: float = 1000   # 首字节时延上限
        self.output_speed_min: float = 20      # 最低输出速度（字符/秒）

    def _resolve_expected_intent(self, case: Dict) -> str:
        """
        解析用例的期望 intent，自动适配当前 ONTOLOGY_LOAD_MODE。

        解析优先级：
          1. expected_intent 为 dict → 取 dict[当前 mode]
          2. expected_intent 为 str  → 忽略（可能与 mode 不对齐）
             改为从 expected_tools[当前 mode][0] 自动推导，
             与 _extract_intent() 的取值逻辑一致。
          3. 若 expected_tools 也为空 → 返回空字符串
        """
        raw = case.get("expected_intent", "")
        if isinstance(raw, dict):
            return raw.get(_ONTOLOGY_LOAD_MODE, "")
        # 自动从 expected_tools[mode][0] 推导，保证与实际工具名一致
        mode_tools = resolve_expected_tools(case)
        return mode_tools[0] if mode_tools else raw

    def judge(self, case: Dict, trace: Dict) -> JudgeResult:
        errors = []

        # 1. Intent 判断（_resolve_expected_intent 自动按 mode 推导期望值）
        actual_intent = trace.get("intent", "")
        expected_intent = self._resolve_expected_intent(case)
        intent_correct = self._check_intent(actual_intent, expected_intent)
        if not intent_correct:
            errors.append({
                "type": "intent_error",
                "mode": _ONTOLOGY_LOAD_MODE,
                "expected": expected_intent,
                "actual": actual_intent,
                "severity": "critical"
            })

        # 2. Tools 判断（按 ONTOLOGY_LOAD_MODE 取对应期望工具名）
        actual_tools = set(trace.get("tools_used", []))
        expected_tools = set(resolve_expected_tools(case))
        tools_correct = actual_tools == expected_tools
        if not tools_correct:
            errors.append({
                "type": "tools_error",
                "mode": _ONTOLOGY_LOAD_MODE,
                "expected": sorted(expected_tools),
                "actual": sorted(actual_tools),
                "severity": "major"
            })

        # 3. SQL 判断
        actual_sql = trace.get("generated_sql", "")
        expected_pattern = case.get("expected_sql_pattern", "")
        sql_correct = bool(re.search(expected_pattern, actual_sql, re.IGNORECASE)) if actual_sql and expected_pattern else False
        if not sql_correct:
            errors.append({
                "type": "sql_error",
                "expected_pattern": expected_pattern,
                "actual_sql": actual_sql,
                "severity": "critical"
            })

        # 4. Fields 判断
        actual_fields = set(trace.get("returned_fields", []))
        expected_fields = set(case.get("expected_fields", []))
        fields_correct = expected_fields.issubset(actual_fields)
        if not fields_correct:
            missing = expected_fields - actual_fields
            errors.append({
                "type": "fields_error",
                "missing_fields": list(missing),
                "severity": "major"
            })

        # 5. 性能判断（TTFB ≤ 1000 ms，输出速度 ≥ 20 字符/秒）
        ttfb_ms = trace.get("ttfb_ms")
        output_speed = trace.get("output_speed_chars_per_sec")
        ttfb_pass = trace.get("ttfb_pass", False)
        output_speed_pass = trace.get("output_speed_pass", False)

        if not ttfb_pass:
            errors.append({
                "type": "perf_ttfb",
                "threshold_ms": 1000,
                "actual_ms": round(ttfb_ms, 1) if ttfb_ms is not None else None,
                "severity": "major"
            })
        if not output_speed_pass:
            errors.append({
                "type": "perf_output_speed",
                "threshold_chars_per_sec": 20,
                "actual_chars_per_sec": round(output_speed, 1) if output_speed is not None else None,
                "severity": "major"
            })

        performance_score = (
            (1.0 if ttfb_pass else 0.0) * 0.5 +
            (1.0 if output_speed_pass else 0.0) * 0.5
        )

        # 6. 工具调用参数校验（仅当用例在当前模式下指定了 expected_tool_args 时）
        tool_args_correct = True
        raw_tool_args = case.get("expected_tool_args")
        if raw_tool_args:
            if isinstance(raw_tool_args, dict) and \
               ("ontology_query" in raw_tool_args or "db_query" in raw_tool_args):
                # 模式键控格式：取当前模式对应值；键缺失则跳过（None → 不校验）
                mode_args = raw_tool_args.get(_ONTOLOGY_LOAD_MODE)
            else:
                # 非模式键控格式（旧格式）：直接使用
                mode_args = raw_tool_args
            if mode_args is not None:
                actual_args = trace.get("tool_call_args", {}).get("args", {})
                tool_args_correct = self._check_tool_args(mode_args, actual_args)
                if not tool_args_correct:
                    errors.append({
                        "type": "tool_args_error",
                        "mode": _ONTOLOGY_LOAD_MODE,
                        "expected": mode_args,
                        "actual": actual_args,
                        "severity": "major"
                    })

        # 7. 错误返回校验（使用 resolve_expected_error 按模式解析；None → 跳过）
        error_correct = True
        mode_error = resolve_expected_error(case)
        if mode_error is not None:
            actual_err = trace.get("actual_error", {})
            actual_code = actual_err.get("error_code", "")
            actual_msg  = actual_err.get("message", "")
            exp_code    = mode_error.get("error_code", "")
            exp_pattern = mode_error.get("message_pattern", "")
            error_correct = (
                actual_code == exp_code and
                (not exp_pattern or bool(re.search(exp_pattern, actual_msg, re.IGNORECASE)))
            )
            if not error_correct:
                errors.append({
                    "type": "error_expectation_mismatch",
                    "mode": _ONTOLOGY_LOAD_MODE,
                    "expected_code": exp_code,
                    "actual_code": actual_code,
                    "severity": "critical"
                })

        # 8. 链式查询校验（仅当用例指定 expected_tool_chain 时）
        chain_correct = True
        expected_tool_chain = case.get("expected_tool_chain")
        if expected_tool_chain:
            chain_correct, chain_errors = self._check_tool_chain(
                expected_tool_chain, trace.get("tool_call_sequence", [])
            )
            if not chain_correct:
                errors.extend(chain_errors)

        # 综合评分（链式正确性纳入准确率）
        # 有 expected_tool_chain 时，chain_correct 替代 tools_correct 参与计分
        if expected_tool_chain:
            accuracy_score = (
                (1 if intent_correct else 0)  * self.intent_weight +
                (1 if chain_correct else 0)   * self.tools_weight +
                (1 if sql_correct else 0)     * self.sql_weight +
                (1 if fields_correct else 0)  * self.fields_weight
            )
        else:
            accuracy_score = (
                (1 if intent_correct else 0) * self.intent_weight +
                (1 if tools_correct else 0)  * self.tools_weight +
                (1 if sql_correct else 0)    * self.sql_weight +
                (1 if fields_correct else 0) * self.fields_weight
            )

        status = "pass" if accuracy_score >= 0.8 and not any(e["severity"] == "critical" for e in errors) else \
                 "fail" if accuracy_score < 0.5 else "partial"

        return JudgeResult(
            case_id=case["case_id"],
            trace_id=trace["trace_id"],
            status=status,
            accuracy_score=accuracy_score,
            performance_score=performance_score,
            intent_correct=intent_correct,
            tools_correct=tools_correct,
            sql_correct=sql_correct,
            fields_correct=fields_correct,
            ttfb_pass=ttfb_pass,
            output_speed_pass=output_speed_pass,
            tool_args_correct=tool_args_correct,
            error_correct=error_correct,
            chain_correct=chain_correct,
            errors=errors
        )

    def _check_tool_chain(
        self,
        expected_chain: List[Dict],
        actual_sequence: List[Dict],
    ) -> tuple[bool, List[Dict]]:
        """
        校验链式工具调用：顺序、工具名、结果传递。

        expected_chain: TestCase.expected_tool_chain 中的 list
        actual_sequence: trace["tool_call_sequence"]，每项 {"tool": str, "args": dict}

        返回 (passed: bool, error_entries: list)
        """
        errors: List[Dict] = []
        mode = _ONTOLOGY_LOAD_MODE

        # ① 步骤数检查
        if len(actual_sequence) < len(expected_chain):
            errors.append({
                "type": "chain_incomplete",
                "expected_steps": len(expected_chain),
                "actual_steps": len(actual_sequence),
                "severity": "critical",
            })
            return False, errors

        # ② 逐步校验工具名称
        for exp_step in expected_chain:
            step_idx = exp_step["step"] - 1
            if step_idx >= len(actual_sequence):
                errors.append({
                    "type": "chain_step_missing",
                    "step": exp_step["step"],
                    "severity": "critical",
                })
                continue

            expected_tool = exp_step.get("tool", {}).get(mode, "")
            actual_tool   = actual_sequence[step_idx].get("tool", "")
            if actual_tool != expected_tool:
                errors.append({
                    "type": "chain_tool_mismatch",
                    "step": exp_step["step"],
                    "expected_tool": expected_tool,
                    "actual_tool":   actual_tool,
                    "severity": "critical",
                })

        # ③ 结果传递校验：step N 的 filters 是否包含 step N-1 的 key_output_field 对应的字段
        for exp_step in expected_chain:
            from_step = exp_step.get("filter_from_step")
            filter_field = exp_step.get("filter_field", "")
            if from_step is None or not filter_field:
                continue

            step_idx = exp_step["step"] - 1
            if step_idx >= len(actual_sequence):
                continue

            actual_args = actual_sequence[step_idx].get("args", {})
            filters = actual_args.get("filters", [])
            filter_fields_used = [f.get("field", "") for f in filters]

            if filter_field not in filter_fields_used:
                errors.append({
                    "type": "chain_result_not_passed",
                    "step": exp_step["step"],
                    "expected_filter_field": filter_field,
                    "actual_filter_fields": filter_fields_used,
                    "hint": f"step {from_step} 的 '{exp_step.get('key_output_field', '')}' 结果应作为 step {exp_step['step']} 的 '{filter_field}' 过滤条件",
                    "severity": "major",
                })

        return len(errors) == 0, errors

    def _check_tool_args(self, expected: dict, actual: dict) -> bool:
        """
        检查实际工具调用参数是否与 expected 一致（只校验 expected 中声明的键）。
        对 list 类型做排序无关比较，对 dict 类型做递归校验。
        """
        for key, exp_val in expected.items():
            act_val = actual.get(key)
            if isinstance(exp_val, dict) and isinstance(act_val, dict):
                if not self._check_tool_args(exp_val, act_val):
                    return False
            elif isinstance(exp_val, list) and isinstance(act_val, list):
                if sorted(str(x) for x in exp_val) != sorted(str(x) for x in act_val):
                    return False
            elif exp_val != act_val:
                return False
        return True

    def _check_intent(self, actual: str, expected: str) -> bool:
        """
        精确匹配 intent 工具名。

        _resolve_expected_intent() 已将 expected 解析为当前模式的实际工具名
        （如 ontology_query 模式的 compute_ads_manage_grid_analysis，
          或 db_query 模式的 data_query_ads_manage_grid_analysis），
        而 _extract_intent() 也直接取第一次工具调用的名称，
        因此两侧格式一致，使用精确匹配即可。

        任一侧为空时视为不匹配。
        """
        if not actual or not expected:
            return False
        return actual.strip().lower() == expected.strip().lower()
```

```python
# utils/analyzer.py
from typing import Dict, List, Any
from dataclasses import dataclass

@dataclass
class ErrorAnalysis:
    error_type: str
    location: str              # 哪个环节/函数
    root_cause: str
    expected_behavior: str
    actual_behavior: str
    suggestion: str

class ErrorAnalyzer:
    def analyze(self, judge_result, trace) -> List[ErrorAnalysis]:
        analyses = []

        for error in judge_result.errors:
            error_type = error["type"]

            if error_type == "intent_error":
                analyses.append(ErrorAnalysis(
                    error_type="意图识别错误",
                    location="IntentClassifier.classify()",
                    root_cause=self._analyze_intent_error(error),
                    expected_behavior=f"识别为 {error['expected']}",
                    actual_behavior=f"实际识别为 {error['actual']}",
                    suggestion="检查意图分类器的训练数据或规则配置"
                ))

            elif error_type == "sql_error":
                analyses.append(ErrorAnalysis(
                    error_type="SQL 生成错误",
                    location="SQLGenerator.generate()",
                    root_cause=self._analyze_sql_error(error, trace),
                    expected_behavior=f"应生成匹配模式的 SQL: {error['expected_pattern']}",
                    actual_behavior=f"实际生成: {error['actual_sql']}",
                    suggestion=self._suggest_sql_fix(error, trace)
                ))

            elif error_type == "tools_error":
                analyses.append(ErrorAnalysis(
                    error_type="工具选择错误",
                    location="ToolSelector.select()",
                    root_cause=f"选择了错误工具集",
                    expected_behavior=f"应使用工具: {error['expected']}",
                    actual_behavior=f"实际使用工具: {error['actual']}",
                    suggestion="检查工具选择器的匹配规则"
                ))

            elif error_type == "fields_error":
                analyses.append(ErrorAnalysis(
                    error_type="字段缺失",
                    location="ResultParser.parse()",
                    root_cause=f"返回结果缺少字段",
                    expected_behavior=f"应包含字段: {error['missing_fields']}",
                    actual_behavior="字段缺失",
                    suggestion="检查 SQL 查询是否包含所有必需字段"
                ))

        return analyses

    def _analyze_intent_error(self, error: Dict) -> str:
        expected = error.get("expected", "")
        actual = error.get("actual", "")

        if "manage_grid" in expected and "grid" not in actual:
            return "未能识别管理网格相关查询"
        elif "chain" in expected and "chain" not in actual:
            return "未能识别产业链相关查询"
        elif "enterprise" in expected and "enterprise" not in actual:
            return "未能识别企业相关查询"
        return "意图分类器未能正确区分查询类型"

    def _analyze_sql_error(self, error: Dict, trace: Dict) -> str:
        actual_sql = error.get("actual_sql", "")
        expected_pattern = error.get("expected_pattern", "")

        if not actual_sql:
            return "SQL 生成器未产生任何 SQL"
        if "WHERE" in expected_pattern and "WHERE" not in actual_sql:
            return "缺少过滤条件 WHERE 子句"
        if "ORDER BY" in expected_pattern and "ORDER" not in actual_sql:
            return "缺少排序 ORDER BY 子句"
        if "LIMIT" in expected_pattern and "LIMIT" not in actual_sql:
            return "缺少限制条数 LIMIT 子句"
        return "SQL 内容与预期模式不匹配"

    def _suggest_sql_fix(self, error: Dict, trace: Dict) -> str:
        actual_sql = error.get("actual_sql", "")
        expected_pattern = error.get("expected_pattern", "")

        suggestions = []
        if "WHERE" in expected_pattern and "WHERE" not in actual_sql:
            suggestions.append("添加 WHERE 子句进行条件过滤")
        if "ORDER BY" in expected_pattern and "ORDER" not in actual_sql:
            suggestions.append("添加 ORDER BY 子句进行排序")
        if "LIMIT" in expected_pattern and "LIMIT" not in actual_sql:
            suggestions.append("添加 LIMIT 子句限制返回条数")
        if "GROUP BY" in expected_pattern and "GROUP" not in actual_sql:
            suggestions.append("添加 GROUP BY 子句进行分组聚合")

        return "; ".join(suggestions) if suggestions else "检查 SQL 生成器的模式匹配规则"
```

### 6.7 CaseFilter 用例筛选器

筛选逻辑：**不同维度之间取 AND，同一维度的多个值之间取 OR**。
`--ids` 支持 [`fnmatch`](https://docs.python.org/3/library/fnmatch.html) 通配符（`*` 匹配任意字符，`?` 匹配单字符）。

```python
# cases/loader.py（与 CaseLoader 同文件）
import fnmatch
from typing import Dict, List, Optional


class CaseFilter:
    """
    多维度用例筛选器。

    筛选维度：
      ids        — 用例 ID 列表，支持 fnmatch 通配符（* / ?）
      categories — category 字段值列表（单轮/多轮/链式）
      tags       — tags 字段值列表（任一标签匹配即选中）
      difficulties — difficulty 字段值列表

    各维度之间取 AND；同一维度多个值之间取 OR；未传维度不过滤。
    """

    @staticmethod
    def apply(
        cases: List[Dict],
        *,
        ids: Optional[List[str]] = None,
        categories: Optional[List[str]] = None,
        tags: Optional[List[str]] = None,
        difficulties: Optional[List[str]] = None,
    ) -> List[Dict]:
        result = cases
        if ids:
            result = [c for c in result if CaseFilter._match_ids(c["case_id"], ids)]
        if categories:
            cats = {c.lower() for c in categories}
            result = [c for c in result if c.get("category", "").lower() in cats]
        if tags:
            tag_set = {t.lower() for t in tags}
            result = [
                c for c in result
                if any(t.lower() in tag_set for t in c.get("tags", []))
            ]
        if difficulties:
            diff_set = {d.lower() for d in difficulties}
            result = [c for c in result if c.get("difficulty", "").lower() in diff_set]
        return result

    @staticmethod
    def _match_ids(case_id: str, patterns: List[str]) -> bool:
        """精确匹配或 fnmatch 通配符匹配（* / ?）。"""
        return any(fnmatch.fnmatch(case_id, pat) for pat in patterns)
```

```python
# ── 错误类型 → 调优建议（与 Judge.errors[*].type 对应）──────────────────
_ERROR_HINTS: dict[str, str] = {
    "intent_error":               "意图识别错误 → 检查工具描述是否清晰区分 query/compute 场景",
    "tools_error":                "工具选择错误 → 检查实际调用工具名与 expected_tools 是否一致",
    "tool_args_error":            "工具参数错误 → 检查 select/filters/dimensions/metrics 的字段名填写规范",
    "sql_error":                  "SQL 生成错误 → 检查 formula 展开 / ORDER BY / WHERE 子句生成是否正确",
    "fields_error":               "返回字段缺失 → 检查 select 中是否遗漏必要字段",
    "chain_incomplete":           "链式调用步骤不足 → Agent 未规划出全部子任务，检查 system prompt 对多步任务的引导",
    "chain_tool_mismatch":        "链式工具名称错误 → 检查各步骤工具路由逻辑",
    "chain_result_not_passed":    "链式结果未向下传递 → 检查 step N 的 filters 是否引用了 step N-1 的输出字段",
    "error_expectation_mismatch": "错误码不符 → 检查 validation 层错误码与 expected_error 是否对齐",
    "perf_ttfb":                  "TTFB 超时 (>1000 ms) → 检查 LLM 接口延迟 / first-token 响应配置",
    "perf_output_speed":          "输出速度过慢 (<20 c/s) → 检查流式输出配置 / 网络带宽",
    "runtime_error":              "运行时异常 → 查看 evaluation_results.json 中的 error 字段定位堆栈",
}


def _print_failure_analysis(all_results: list) -> None:
    """
    对本次运行中状态为 fail / partial / error 的用例进行分类汇总。
    按错误类型分组，每组显示受影响用例 ID（最多 5 个）及调优建议。
    """
    non_pass = [r for r in all_results if r.get("status") in ("fail", "partial", "error")]
    if not non_pass:
        print("\n[分析] 全部用例通过，无需调优。")
        return

    error_groups: dict[str, list[str]] = defaultdict(list)
    for r in non_pass:
        errors = r.get("errors", [])
        if errors:
            for err in errors:
                error_groups[err.get("type", "unknown")].append(r["case_id"])
        else:
            # status=error 但无 errors 列表（运行时异常）
            error_groups["runtime_error"].append(r["case_id"])

    print(f"\n{'─' * 58}")
    print(f"[失败分析] 共 {len(non_pass)} 个用例未通过，错误分布：")
    for etype, ids in sorted(error_groups.items(), key=lambda x: -len(x[1])):
        unique_ids = list(dict.fromkeys(ids))   # 去重保序
        display = unique_ids[:5]
        suffix = f"… 共 {len(unique_ids)} 个" if len(unique_ids) > 5 else ""
        hint = _ERROR_HINTS.get(etype, "（暂无对应建议）")
        print(f"  [{etype}] × {len(unique_ids):2d}  {display}{suffix}")
        print(f"           ↳ {hint}")
    print(f"{'─' * 58}")
    print(f"[提示] 调优后执行 --retry-failed ./reports/failed_cases.json 只重跑以上用例。")


async def _build_eval_worker() -> EvalDataCloudWorker:
    """
    初始化 EvalDataCloudWorker：
      1. 连接 Redis（与生产配置一致）
      2. 注册插件（InitDataCloudDigitalEmployeePlugin 加载数字员工配置）
      3. start_heartbeat()：bootstrap datacloud_analysis SDK + 加载 agent configs
    """
    worker = EvalDataCloudWorker(
        model_name=os.environ.get("DATACLOUD_LLM_REASONING_MODEL"),
        api_key=os.environ.get("OPENAI_API_KEY"),
        base_url=os.environ.get("OPENAI_BASE_URL"),
    )

    import redis.asyncio as aioredis
    from by_framework.core.extensions import PluginRegistry

    redis_client = aioredis.from_url(
        f"redis://{os.environ.get('DATACLOUD_GATEWAY_REDIS_HOST', 'localhost')}:"
        f"{os.environ.get('DATACLOUD_GATEWAY_REDIS_PORT', '6379')}/"
        f"{os.environ.get('DATACLOUD_GATEWAY_REDIS_DB', '0')}",
        password=os.environ.get("DATACLOUD_GATEWAY_REDIS_PASSWORD"),
        decode_responses=False,
    )
    worker.redis = redis_client

    plugin_registry = PluginRegistry(
        plugin_list=[
            InitDataCloudDigitalEmployeePlugin(),
            RecommendedQuestionsPlugin(),
        ]
    )
    worker.plugin_registry = plugin_registry
    await worker.start_heartbeat()
    return worker


async def main() -> None:
    parser = argparse.ArgumentParser(description="准确率 & 性能评测")
    parser.add_argument("--cases",    type=str, required=True,          help="评测用例 JSON 文件路径")
    parser.add_argument("--output",   type=str, default="./reports",    help="评测报告输出目录")
    parser.add_argument("--agent-id", type=str, default="10000587",     help="数字员工 Agent ID")

    # ── 用例筛选（各维度之间 AND，同维度多值之间 OR）──────────────────
    parser.add_argument("--ids",        type=str, default=None,
                        help="指定用例 ID，逗号分隔，支持 fnmatch 通配符，如 TC_Q_*,TC_Chain_001")
    parser.add_argument("--category",   type=str, default=None,
                        help="按 category 筛选，逗号分隔，如 single_turn,chain_query")
    parser.add_argument("--tags",       type=str, default=None,
                        help="按标签筛选（任一匹配即选中），逗号分隔，如 compute,chain_query")
    parser.add_argument("--difficulty", type=str, default=None,
                        help="按难度筛选，逗号分隔，如 easy,medium,hard")

    # ── 增量调优 ──────────────────────────────────────────────────────
    parser.add_argument(
        "--retry-failed", type=str, default=None, metavar="FAILED_JSON",
        help=(
            "只重跑上次失败的用例。"
            "传入上次评测 --output 目录下自动生成的 failed_cases.json 路径，"
            "如 ./reports/failed_cases.json。"
            "与 --ids/--category/--tags/--difficulty 互斥。"
        ),
    )
    args = parser.parse_args()

    # 1. 初始化 Worker
    print(f"[评测] 初始化 EvalDataCloudWorker，agent_id={args.agent_id}...")
    worker = await _build_eval_worker()

    # 2. 配置
    config = EvaluationConfig(
        worker=worker,
        agent_id=args.agent_id,
        cases_file=args.cases,
        output_dir=args.output,
        performance_threshold_ms=5000,
    )

    # 3. 加载并筛选用例
    def _split(val: str | None) -> list[str] | None:
        return [x.strip() for x in val.split(",") if x.strip()] if val else None

    loader = CaseLoader()
    cases = loader.load_cases(args.cases)
    retry_mode = False

    if args.retry_failed:
        # ── 增量重跑：从 failed_cases.json 读取上次失败的 case_id 列表 ──
        retry_mode = True
        with open(args.retry_failed, encoding="utf-8") as fh:
            failed_info = json.load(fh)
        prev_failed_ids: list[str] = failed_info.get("failed_ids", [])
        if not prev_failed_ids:
            print("[评测] failed_cases.json 中无失败用例，无需重跑。")
            await worker.redis.aclose()
            return
        cases = CaseFilter.apply(cases, ids=prev_failed_ids)
        print(
            f"[评测] 重跑模式：上次失败 {len(prev_failed_ids)} 个，"
            f"本次 cases 文件中匹配到 {len(cases)} 个"
        )
    else:
        # ── 普通模式：多维筛选 ──
        cases = CaseFilter.apply(
            cases,
            ids=_split(args.ids),
            categories=_split(args.category),
            tags=_split(args.tags),
            difficulties=_split(args.difficulty),
        )

    if not cases:
        print("[评测] 警告：筛选后无匹配用例，请检查筛选条件。")
        await worker.redis.aclose()
        return
    print(f"[评测] 共 {len(cases)} 个用例待评测")

    # 4. 执行评测
    runner = EvaluationRunner(config)
    all_results: list[dict] = []
    for case in cases:
        print(f"[评测] 运行用例: {case['case_id']}")
        try:
            if case["category"] in ("single_turn", "chain_query"):
                result = await runner.run_single_turn(case)
            elif case["category"] == "multi_turn":
                result = await runner.run_multi_turn(case)
            else:
                raise ValueError(f"未知 category: {case['category']}")
            all_results.append(result)
            ttfb = result.get("ttfb_ms")
            spd  = result.get("output_speed_chars_per_sec")
            print(
                f"[评测] {case['case_id']}: {result.get('status')} "
                f"| 准确率: {result.get('accuracy_score', 0):.2%} "
                f"| TTFB: {f'{ttfb:.0f}ms' if ttfb is not None else 'N/A'} "
                f"| 速度: {f'{spd:.1f}c/s' if spd is not None else 'N/A'}"
            )
        except Exception as exc:
            print(f"[评测] {case['case_id']} 执行失败: {exc}")
            all_results.append({
                "case_id": case["case_id"],
                "status": "error",
                "error": str(exc),
                "errors": [],
            })

    # 5. 保存完整结果
    output_path = Path(args.output)
    output_path.mkdir(parents=True, exist_ok=True)
    results_file = output_path / "evaluation_results.json"
    with open(results_file, "w", encoding="utf-8") as f:
        json.dump(all_results, f, ensure_ascii=False, indent=2)

    # 6. 更新 failed_cases.json（始终只保留本次仍未通过的 ID）
    failed_ids_now = [
        r["case_id"] for r in all_results
        if r.get("status") in ("fail", "partial", "error")
    ]
    failed_cases_file = output_path / "failed_cases.json"
    with open(failed_cases_file, "w", encoding="utf-8") as f:
        json.dump(
            {
                "generated_at": datetime.datetime.now().isoformat(timespec="seconds"),
                "cases_file":   args.cases,
                "run_total":    len(all_results),
                "failed_count": len(failed_ids_now),
                "failed_ids":   failed_ids_now,
            },
            f,
            ensure_ascii=False,
            indent=2,
        )

    # 7. 打印汇总
    passed  = sum(1 for r in all_results if r.get("status") == "pass")
    failed  = sum(1 for r in all_results if r.get("status") == "fail")
    partial = sum(1 for r in all_results if r.get("status") == "partial")
    ttfb_pass_count = sum(1 for r in all_results if r.get("ttfb_pass"))
    spd_pass_count  = sum(1 for r in all_results if r.get("output_speed_pass"))
    total = len(all_results) or 1

    print(f"\n{'=' * 58}")
    if retry_mode:
        fixed_count = len([r for r in all_results if r.get("status") == "pass"])
        print(
            f"[评测汇总] 重跑 {len(all_results)} 个 | "
            f"本次修复: +{fixed_count} ✓ | 仍未通过: {len(failed_ids_now)}"
        )
    else:
        print(
            f"[评测汇总] 总用例: {len(all_results)} | "
            f"通过: {passed} | 失败: {failed} | 部分: {partial}"
        )
    print(
        f"[评测汇总] TTFB 通过: {ttfb_pass_count}/{total} ({ttfb_pass_count/total:.1%}) | "
        f"输出速度通过: {spd_pass_count}/{total} ({spd_pass_count/total:.1%})"
    )
    print(f"[评测汇总] 详细结果 → {results_file}")
    print(f"[评测汇总] 失败清单 → {failed_cases_file}")
    print(f"{'=' * 58}")

    # 8. 自动失败分析（有失败时打印根因分类 + 调优建议）
    _print_failure_analysis(all_results)

    # 9. 关闭 Redis 连接
    await worker.redis.aclose()


if __name__ == "__main__":
    asyncio.run(main())
```

### 8.6 增量调优工作流

完整调优循环：**全量跑 → 分析失败 → 调优 → 只重跑失败 → 重复**，直到全部通过为止。

#### 工作流图

```
┌──────────────────────────────────────────────────────────────────┐
│ 1. 全量评测（首次）                                                │
│    python -m evaluation.main --cases cases.json --output reports  │
└──────────────┬───────────────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 2. 自动输出                                                        │
│    ① reports/eval_result_<timestamp>.json   — 完整结果            │
│    ② reports/failed_cases.json              — 失败 ID 列表        │
│    ③ 终端 [分析] 块                          — 按错误类型分组建议  │
└──────────────┬───────────────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 3. 用户阅读 [分析] 建议，调整：                                    │
│    · 工具描述（intent / tool_args 错）                            │
│    · Prompt / few-shot 示例（sql_error / fields_error）           │
│    · Validation 层错误码（error_expectation_mismatch）            │
└──────────────┬───────────────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 4. 增量重跑（仅失败用例）                                          │
│    python -m evaluation.main --cases cases.json                   │
│        --retry-failed reports/failed_cases.json                   │
│        --output reports                                           │
└──────────────┬───────────────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 5. 重跑结果                                                        │
│    [评测汇总] 重跑 10 个 | 本次修复: +7 ✓ | 仍未通过: 3           │
│    failed_cases.json 自动更新为「仍未通过」的 3 个 ID             │
└──────────────┬───────────────────────────────────────────────────┘
               │ 仍有失败
               ▼
          回到步骤 3（再次调优）
               │ 全部通过
               ▼
          failed_cases.json 写入空列表，[分析] 输出"全部用例通过"
```
