# 本体调用优化设计

## 问题背景

### 问题1：system_prompt 未生效
当前在 `worker.py:587` 中传入了 `prompts_dict`（包含 `system_prompt` 和 `task_prompt`）到 `create_agent` 的 `prompts_overwrite` 参数，但是：
- `agent.py:48` 注释中标记 `prompts_overwrite` 为 legacy，已被忽略
- `agent.py:71` 实际传入 `_create_deep_agent` 的 `system_prompt` 参数为 None
- 导致业务配置的 system_prompt 无法生效

示例配置（未生效）：
```python
prompts_overwrite = {
    'system_prompt': '我是亦庄产业大脑数字员工，我可以提供亦庄产业链分析、管理网格综合分析、企业综合分析表服务。',
    'task_prompt': '请根据以下原则来处理问题。\n# 单一技能优先原则：...'
}
```

### 问题2：tools_dict 直接注入到 agent
当前在 `worker.py:588` 中，`tools_dict` 直接注入到 agent 中：
```python
create_agent(
    prompts_overwrite=prompts_dict,
    tools=tools_dict,  # 直接注入
)
```

在 `agent.py:122` 中，这些工具被添加到 `all_tools`：
```python
all_tools = register_all_tools()
if tools:
    all_tools.extend(tools.values())  # 直接注册
```

**问题**：
1. 工具数量膨胀：每个业务域都会注册大量视图/对象工具，导致 LLM 上下文过大
2. 维护困难：业务视图变更需要重新构建 agent graph
3. 缺乏灵活性：无法根据运行时上下文动态选择可用工具

### 问题3：知识注入不完整
当前 `KnowledgeInjectionMiddleware` 仅注入了简单的术语匹配信息：
```python
# 当前实现（knowledge_injection.py:78-86）
schema_lines = ["<ontology_context>"]
schema_lines.append("相关本体术语：")
for match in term_matches[:5]:
    term_name = match.get("term_name", "")
    term_type = match.get("term_type_code", "")
    match_type = match.get("match_type", "")
    schema_lines.append(f"- {term_name} ({term_type}) [匹配类型: {match_type}]")
```

**缺少关键信息**：
- 对象的完整结构（属性列表、属性类型）
- 对象间的关系
- 对象支持的 action
- 属性的编码和别名（用于参数抽取）

## 设计目标

1. **system_prompt 生效**：确保 worker 传入的业务自定义 system_prompt 能够正确覆盖默认 prompt
2. **工具动态化**：OBJECT/VIEW 类型不再直接注入 agent，而是通过知识注入传递元数据；其他类型（AGENT、FUNCTION 等）保持原有注册方式
3. **知识注入增强**：注入对象的完整元数据（名称、编码、描述、属性、关系、action）
4. **统一调用入口**：所有本体操作统一通过 `query_objects` 和 `execute_action` 两个核心工具
5. **兼容性保证**：非 OBJECT/VIEW 类型的工具（如 AGENT 委托）保持原有功能不变

def create_agent(
    *,
    model: str | None = None,
    api_key: str | None = None,
    base_url: str | None = None,
    temperature: float = 0.7,
    locale: str | None = None,
    system_prompt: str | None = None,
    prompts_overwrite: dict[str, Any] | None = None,
    tools: dict[str, Any] | None = None,
) -> Any:
    """Create a deep agent for DataCloud using Deep Agents SDK.

    Args:
        model: LLM model name
        api_key: API key for LLM provider
        base_url: Base URL for LLM provider
        temperature: LLM temperature
        locale: Locale for agent (default: zh_CN)
        system_prompt: Custom system prompt
        prompts_overwrite: Prompt overrides from worker (优先级最高)
            - system_prompt: 覆盖默认 system prompt
            - task_prompt: 附加任务指导（将追加到 system_prompt）
        tools: Additional tools to register

    Returns:
        Compiled agent graph
    """
    # Resolve locale
    resolved_locale = locale or os.getenv("DATACLOUD_AGENT_LOCALE", "zh_CN")
    supported = get_supported_locales()
    if resolved_locale not in supported:
        logger.warning(
            "create_agent: locale=%r is not supported (supported: %s). Falling back to zh_CN.",
            resolved_locale,
            supported,
        )
        resolved_locale = "zh_CN"

    # 优先级: prompts_overwrite['system_prompt'] > system_prompt > default
    final_system_prompt = system_prompt
    task_prompt = None
    
    if prompts_overwrite:
        if "system_prompt" in prompts_overwrite:
            final_system_prompt = prompts_overwrite["system_prompt"]
            logger.info("create_agent: using system_prompt from prompts_overwrite")
        if "task_prompt" in prompts_overwrite:
            task_prompt = prompts_overwrite["task_prompt"]
            logger.info("create_agent: using task_prompt from prompts_overwrite")

    return _create_deep_agent(
        model=model,
        api_key=api_key,
        base_url=base_url,
        temperature=temperature,
        locale=resolved_locale,
        system_prompt=final_system_prompt,  # 传递解析后的 system_prompt
        task_prompt=task_prompt,  # 新增参数
        tools=tools,
    )
```

def _create_deep_agent(
    *,
    model: str | None = None,
    api_key: str | None = None,
    base_url: str | None = None,
    temperature: float = 0.7,
    locale: str = "zh_CN",
    system_prompt: str | None = None,
    task_prompt: str | None = None,  # 新增
    tools: dict[str, Any] | None = None,
) -> Any:
    """Create agent using Deep Agents SDK.

    Args:
        model: LLM model name
        api_key: API key
        base_url: Base URL
        temperature: Temperature
        locale: Locale
        system_prompt: System prompt (优先使用此参数)
        task_prompt: Additional task guidance (追加到 system_prompt)
        tools: Additional tools

    Returns:
        Compiled agent
    """
    import pathlib

    from deepagents import create_deep_agent
    from datacloud_analysis.backend import create_datacloud_backend
    from datacloud_analysis.context import DatacloudContext
    from datacloud_analysis.middlewares import (
        DatacloudOutputMiddleware,
        KnowledgeInjectionMiddleware,
        WorkspaceInitMiddleware,
    )
    from datacloud_analysis.subagents import CODE_EXECUTOR_SUBAGENT
    from datacloud_analysis.tools.registry import register_all_tools

    logger.info("create_agent: Using Deep Agents SDK (locale=%s)", locale)

    # Register OQL tools (emit_result injected by DatacloudOutputMiddleware)
    all_tools = register_all_tools()
    if tools:
        all_tools.extend(tools.values())

    # Build system prompt: default + custom + task_prompt
    final_system_prompt = system_prompt or _build_default_system_prompt(locale)
    
    if task_prompt:
        final_system_prompt = f"{final_system_prompt}\n\n# 任务处理指导\n{task_prompt}"
        logger.info("create_agent: appended task_prompt to system_prompt")

    # Workspace dir for backend
    workspace_dir = os.getcwd()

    # DatacloudBackend — limits file operations to workspace_dir
    backend = create_datacloud_backend(workspace_dir)

    # Built-in skills directory
    builtin_skills_dir = str(pathlib.Path(__file__).parent / "skills" / "builtin")
    skill_sources = [builtin_skills_dir]

    # Middleware stack
    middlewares = [
        KnowledgeInjectionMiddleware(),
        DatacloudOutputMiddleware(),
        WorkspaceInitMiddleware(
            workspace_dir=workspace_dir,
            agent_name="DataCloud Agent",
        ),
    ]

    # Resolve checkpointer
    checkpointer: Any = None
    try:
        from datacloud_analysis.session.checkpointer import get_checkpointer

        checkpointer = get_checkpointer()
        logger.info("create_agent: using PG checkpointer")
    except RuntimeError:
        logger.warning(
            "create_agent: checkpointer not initialized — running without checkpointing. "
            "Call `await bootstrap.setup()` before create_agent() to enable interrupt/resume."
        )

    # Resolve model
    resolved_model: Any
    if base_url:
        from langchain_openai import ChatOpenAI

        resolved_model = ChatOpenAI(
            model=model or "gpt-4o",
            base_url=base_url,
            api_key=api_key or "sk-placeholder",
            temperature=temperature,
        )
        logger.info(
            "create_agent: using ChatOpenAI with custom base_url=%s model=%s",
            base_url,
            model,
        )
    else:
        resolved_model = model or "claude-sonnet-4-6"

    # Create compiled agent with Deep Agents SDK
    compiled = create_deep_agent(
        model=resolved_model,
        tools=all_tools,
        system_prompt=final_system_prompt,  # 使用构建后的完整 prompt
        middleware=middlewares,
        context_schema=DatacloudContext,
        backend=backend,
        skills=skill_sources,
        subagents=[CODE_EXECUTOR_SUBAGENT],
        checkpointer=checkpointer,
    )

    return compiled
```

##### 修改 `_build_query_tool` 方法签名

不再绑定 `query_func` 参数，改为传递工具元数据（tool_code、tool_type、tool_attributes），后续由 `query_objects` 和 `execute_action` 统一执行。

```python
def _build_query_tool(
    self,
    *,
    tool_name: str,
    tool_desc: str = "",
    tool_code: str = "",
    tool_type: str = "query",  # query/action
    tool_attributes: dict[str, Any] | None = None,
) -> dict[str, Any]:
    """构建工具元数据，不再绑定具体的查询函数。
    
    Args:
        tool_name: 工具名称，从 rel.get('resourceName', resource_name) 获取
        tool_desc: 工具描述，从 rel.get('resourceDesc', '') 获取
        tool_code: 工具编码，从 rel.get('resourceCode', '') 获取
        tool_type: 工具类型（query/action）
        tool_attributes: 工具属性，包含：
            - type: rel.get('resourceBizType', 'OBJECT')  # OBJECT/VIEW/AGENT/FUNCTION
            - sub_type: rel.get('resourceType', 'DB_TABLE')  # DB_TABLE/API/DOC
    
    Returns:
        工具元数据字典，包含 tool_code 用于后续通过 OntologyLoader 获取对象定义
    """
    tool_attributes = tool_attributes or {}
    
    metadata = {
        "tool_name": tool_name,
        "tool_desc": tool_desc,
        "tool_code": tool_code,  # 对应 object_code，可通过 ontology_loader.get_ontology_class(tool_code) 获取
        "tool_type": tool_type,
        "tool_attributes": tool_attributes,
    }
    
    # 兼容场景：如果 type 不是 OBJECT/VIEW，保留原有的工具函数
    # 这样 AGENT、FUNCTION 等类型的工具仍然可以直接注册到 agent
    if tool_attributes.get("type") not in ("OBJECT", "VIEW"):
        # 为其他类型构建实际的工具函数（保留原有逻辑）
        # 注意：这里需要根据实际情况传入 query_func 或其他函数
        # 暂时返回占位符，实际使用时需要在调用处传入函数
        metadata["tool_func"] = None  # 调用处需要设置实际函数
    
    return metadata
```

#### 2.2 修改 Worker 提取挂载对象列表

修改文件：`examples/e_commerce_demo/backend/datacloud_service/worker.py`

```python
def _build_graph(self, prompts_dict: dict | None = None, tools_dict: dict | None = None) -> Any:
    """Instantiate the datacloud-analysis compiled graph with dynamic context."""
    
    # 从 tools_dict 提取挂载的对象/视图列表，并分离其他类型工具
    mounted_objects = []
    other_tools = {}
    
    if tools_dict:
        for tool_name, tool_meta in tools_dict.items():
            # tool_meta 现在是字典，包含 tool_code 和 tool_attributes
            if isinstance(tool_meta, dict):
                tool_attributes = tool_meta.get("tool_attributes", {})
                tool_type = tool_attributes.get("type", "")
                
                # 如果是 OBJECT 或 VIEW 类型，提取到 mounted_objects
                if tool_type in ("OBJECT", "VIEW"):
                    tool_code = tool_meta.get("tool_code", "")
                    if tool_code and tool_code not in mounted_objects:
                        mounted_objects.append(tool_code)
                else:
                    # 其他类型（如 AGENT、FUNCTION 等）保留为工具
                    other_tools[tool_name] = tool_meta.get("tool_func")
    
    logger.info(
        "Extracted from tools_dict: mounted_objects=%s other_tools=%s", 
        mounted_objects, 
        list(other_tools.keys())
    )
    
    # 传递挂载对象列表和其他工具给 agent
    return create_agent(
        model=self.model_name,
        api_key=self.api_key,
        base_url=self.base_url,
        prompts_overwrite=prompts_dict,
        tools=other_tools,  # 仅传递非 OBJECT/VIEW 类型的工具
        mounted_objects=mounted_objects,  # 新增参数
    )
```

```python
async def _inject_mounted_objects(self) -> Optional[str]:
    """注入 agent 挂载的所有对象/视图（阶段1实现）。"""
    try:
        router = get_oql_router()
        ontology_loader = router.ontology_loader
        
        # 获取要注入的对象列表
        if self.mounted_objects:
            object_codes = self.mounted_objects
        else:
            # 如果未指定，获取所有对象
            all_classes = ontology_loader.get_ontology_classes()
            object_codes = [cls.object_code for cls in all_classes]
        
        if not object_codes:
            return None
        
        # 获取所有关系（一次性查询，避免重复）
        all_relations = ontology_loader.get_ontology_relations()
        
        schema_lines = ["<ontology_context>"]
        schema_lines.append("## 可用对象类型（object_type）\n")
        
        for idx, object_code in enumerate(object_codes[:10], 1):  # 最多10个对象
            try:
                # 1. 获取对象定义
                ontology_class = ontology_loader.get_ontology_class(object_code)
                
                # 2. 格式化对象 Schema（包含属性、关系、动作）
                formatted = self._format_object_schema(
                    ontology_class, 
                    all_relations,
                    ontology_loader
                )
                schema_lines.append(formatted)
                
                if idx < len(object_codes) and idx < 10:
                    schema_lines.append("\n---\n")
            except Exception as e:
                logger.warning("Failed to load object %s: %s", object_code, e)
                continue
        
        schema_lines.append("\n</ontology_context>")
        return "\n".join(schema_lines)
        
    except Exception as e:
        logger.warning("Failed to inject mounted objects: %s", e)
        return None

def _format_object_schema(
    self, 
    ontology_class: OntologyClass,
    all_relations: list[OntologyRelation],
    ontology_loader
) -> str:
    """格式化单个对象的完整 Schema。"""
    lines = []
    
    # 1. 对象基本信息（名称、编码、描述）
    lines.append(f"### 对象类型：{ontology_class.object_name}")
    lines.append(f"**object_type**: `{ontology_class.object_code}`")
    
    # 判断是否为视图
    is_view = "view" in ontology_class.object_code.lower() or "ads_" in ontology_class.object_code
    if is_view:
        lines.append("**类型**: 预定义视图")
    
    if ontology_class.description:
        lines.append(f"**描述**: {ontology_class.description}")
    lines.append("")
    
    # 2. 属性列表（属性名称、属性编码、类型、说明、别名）
    if ontology_class.fields:
        lines.append("#### 属性列表")
        lines.append("| 属性名称 | 属性编码 | 类型 | 说明 | 别名 |")
        lines.append("|---------|---------|------|------|------|")
        
        for field in ontology_class.fields[:10]:  # 最多显示10个属性
            field_name = field.field_name
            field_code = field.field_code
            field_type = field.field_type
            description = field.description or ""
            aliases = ", ".join(field.aliases[:3]) if field.aliases else ""  # 最多3个别名
            
            lines.append(f"| {field_name} | {field_code} | {field_type} | {description} | {aliases} |")
        
        lines.append("")
    
    # 3. 关联关系（仅对象，视图不显示关系）
    if not is_view:
        object_relations = self._filter_object_relations(
            ontology_class.object_code, 
            all_relations,
            ontology_loader
        )
        if object_relations:
            lines.append("#### 关联关系")
            for rel in object_relations[:5]:  # 最多显示5个关系
                source_name = rel["source_name"]
                target_name = rel["target_name"]
                rel_type = rel["relation_type"]
                desc = rel["description"] or rel["relation_name"]
                
                lines.append(f"- **{source_name} → {target_name}** ({rel_type}): {desc}")
            lines.append("")
    
    # 4. 可用动作（动作名称、动作编码、参数列表）
    if ontology_class.actions:
        lines.append("#### 可用动作")
        for action in ontology_class.actions[:5]:  # 最多显示5个动作
            action_name = action.action_name
            action_code = action.action_code
            description = action.description or ""
            
            lines.append(f"- **{action_name}** (`{action_code}`)")
            if description:
                lines.append(f"  - 描述: {description}")
            
            # 参数列表
            if action.params:
                params_str = []
                for param in action.params[:5]:  # 最多显示5个参数
                    param_code = param.param_code
                    param_type = param.param_type
                    required = "必填" if param.required else "可选"
                    params_str.append(f"`{param_code}` ({param_type}, {required})")
                
                if params_str:
                    lines.append(f"  - 参数: {', '.join(params_str)}")
            
            lines.append("")
    
    return "\n".join(lines)

def _filter_object_relations(
    self,
    object_code: str,
    all_relations: list[OntologyRelation],
    ontology_loader
) -> list[dict]:
    """过滤与指定对象相关的关系。"""
    relations = []
    
    for rel in all_relations:
        if rel.source_class == object_code or rel.target_class == object_code:
            # 获取源和目标对象的名称
            try:
                source_obj = ontology_loader.get_ontology_class(rel.source_class)
                source_name = source_obj.object_name
            except:
                source_name = rel.source_class
            
            try:
                target_obj = ontology_loader.get_ontology_class(rel.target_class)
                target_name = target_obj.object_name
            except:
                target_name = rel.target_class
            
            relations.append({
                "source_class": rel.source_class,
                "source_name": source_name,
                "target_class": rel.target_class,
                "target_name": target_name,
                "relation_type": rel.relation_type,
                "relation_name": rel.relation_name,
                "description": rel.description,
            })
    
    return relations
```

#### 3.1 知识注入时机和去重机制

##### 注入时机
知识注入发生在 **每次 LLM 调用前**，由 `KnowledgeInjectionMiddleware.awrap_model_call` 触发。

**注入流程**：
1. 用户发送消息 → LangGraph 准备调用 LLM
2. `KnowledgeInjectionMiddleware` 拦截请求
3. 提取用户最后一条消息作为查询
4. 检索相关本体知识
5. 将知识追加到 system_message
6. 继续调用 LLM

**注入策略**（分两阶段实现）：

**阶段2（已实现）：默认注入 agent 挂载的所有对象/视图**
- 从 worker 传入的配置中获取 agent 挂载的对象/视图列表
- 将所有挂载的对象/视图元数据注入到上下文
- 不做意图过滤，全量注入

**阶段3（本阶段实现）：基于意图的精细化注入**
- 根据用户问题分析意图
- 仅注入与意图相关的对象/视图
- 包含相关的属性、动作、关系
- 实现接口 `_filter_relevant_objects(query, all_objects)`

##### 去重机制

**问题**：每次 LLM 调用都会注入知识，可能导致上下文中出现重复的本体信息。

**解决方案**：
1. **中间件层去重**：在 `KnowledgeInjectionMiddleware` 中检查 system_message 是否已包含 `<ontology_context>` 标签
   - 如果已存在，跳过注入
   - 如果不存在，执行注入

2. **会话级缓存**（可选优化）：
   - 在 context 中维护 `injected_objects: set[str]`
   - 记录已注入的 object_type
   - 避免重复注入相同对象

    def _get_object_relations(self, object_code: str, ontology_loader) -> list[dict]:
        """
        获取对象的关联关系。
        
        Args:
            object_code: 对象编码
            ontology_loader: OntologyLoader 实例
            
        Returns:
            关系列表
        """

            # 获取所有关系
            all_relations = ontology_loader._relations
            
            for rel in all_relations:

                    # 获取源和目标对象的名称

                    target_name = ""

                        source_obj = ontology_loader.get_ontology_class(rel.source_class)

                    try:
