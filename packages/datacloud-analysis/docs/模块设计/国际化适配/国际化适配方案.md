# 国际化适配方案

> 版本：v1.0  
> 日期：2026-05-12  
> 范围：`datacloud-analysis` SDK 及 `byclaw-data` worker 入口

---

## 1. 背景与目标

用户通过 Gateway 提问时，`header_metadata` 中携带 `language` 字段（如 `en_US` / `zh-CN`），
当前该字段未被提取，系统始终以 `zh_CN` 作为 locale 回退，导致英文用户收到中文回答。

**目标**：将 `header_metadata.language` 完整传递到 LLM system prompt 选择层，
使提问和回答均适配用户语言。

---

## 2. 现状分析

### 2.1 语言字段来源

```python
header_metadata = {
    "channelExtension": {"channelType": "web"},
    "agentId": "10000198",
    "language": "en_US",   # ← 目标字段
    ...
}
```

`header_metadata` 由 `byclaw-data/worker.py` 从 `command.header.metadata` 提取（line 944）。

### 2.2 当前 locale 传递链路（有断点）

```
header_metadata.language
        ↓  ← 【断点1】从未被读取
context.locale              ← getattr 默认空串
        ↓
configurable["locale"]      ← 回退 "zh_CN"（worker.py line 1449）
        ↓  ← 【断点2】configurable["locale"] 未被消费
graph_builder._build_*()    ← 直接读 os.getenv("DATACLOUD_AGENT_LOCALE", "zh_CN")
        ↓
get_system_prompt(locale)   ← i18n/prompts.py，已支持 zh_CN / en_US
```

### 2.3 locale 规范化

| 前端传值 | 规范化后 |
|---------|---------|
| `en_US` | `en_US` |
| `zh-CN` | `zh_CN` |
| `zh_CN` | `zh_CN` |
| 空 / 未知 | `zh_CN`（fallback） |

前端使用连字符 `zh-CN`，内部统一使用下划线 `zh_CN`，需在入口处规范化。

### 2.4 `OntologyAgent`（无 Gateway 路径）

`OntologyAgent.ask()` 已有 `locale` 参数（`ontology_agent.py` line 256），
并通过 `prompts_overwrite={"locale": effective_locale}` 写入初始 state（line 441）。
但 `graph_builder` 未从 `prompts_overwrite["locale"]` 读取，仍走环境变量。

## 3. 改动范围

### 3.1 `byclaw-data/worker.py`（byclaw-all 仓库）

**文件**：`byclaw-all/byclaw-data/src/byclaw_data/worker.py`

在 `header_metadata` 提取后，读取 `language` 字段并规范化，写入 `context.locale`。

新增辅助函数（模块级）：

```python
_LOCALE_ALIAS: dict[str, str] = {
    "zh-cn": "zh_CN",
    "zh_cn": "zh_CN",
    "en-us": "en_US",
    "en_us": "en_US",
}
_SUPPORTED_LOCALES: frozenset[str] = frozenset({"zh_CN", "en_US"})
_FALLBACK_LOCALE = "zh_CN"

def _normalize_locale(raw: str) -> str:
    """将前端 language 字段规范化为内部 locale 码。"""
    if not raw:
        return _FALLBACK_LOCALE
    normalized = _LOCALE_ALIAS.get(raw.lower(), raw)
    return normalized if normalized in _SUPPORTED_LOCALES else _FALLBACK_LOCALE
```

在 `header_metadata` 提取之后（约 line 946）新增：

```python
_raw_language = str(header_metadata.get("language") or "").strip()
context.locale = _normalize_locale(_raw_language)
```

在 `graph_input` 构建处（非 ResumeCommand 路径）写入 `prompts_overwrite`：

```python
graph_input["prompts_overwrite"] = {
    "locale": str(getattr(context, "locale", "") or "zh_CN"),
}
```

> **前提**：`AgentContext` 需支持 `locale` 属性。若无该字段，在 `AgentContext` 中新增
> `locale: str = "zh_CN"` 即可（向后兼容）。

### 3.2 `datacloud-analysis/graph_builder.py`

**文件**：`packages/datacloud-analysis/src/datacloud_analysis/orchestration/graph_builder.py`

`_build_legacy_graph`（line 193）和 `_build_prebuilt_graph`（line 422）两处，
从 `prompts_overwrite["locale"]` 读取 locale，优先级高于环境变量。

```python

overwrite = prompts_overwrite or {}

# 改后：
overwrite = prompts_overwrite or {}

```

### 3.3 `datacloud-analysis/execution/node.py`

**文件**：`packages/datacloud-analysis/src/datacloud_analysis/orchestration/execution/node.py`

`execution_node` 函数（约 line 258），从 state 的 `prompts_overwrite` 读取 locale：


## 4. 涉及下游 SDK 的 language 参数传递

### 4.1 `datacloud-data` SDK — 已支持，需补充传递

**包路径**：`packages/datacloud-data`

#### 现状（已实现）

`datacloud-data` 已完整支持 `language` 字段，**无需修改 SDK 本身**：

- `RequestContext.language`（`context.py`）：存储语言码，默认 `"zh-CN"`
- `InvocationContext(**kwargs)`：构造时接受 `language` 或 `locale` 关键字参数，
  内部调用 `normalize_language()` 规范化
- `i18n.py` 的 `normalize_language()`：支持 `zh-CN`、`en_US` 及常见别名
- `get_current_language()`：在 SDK 内部任意位置获取当前请求语言

```python
# 已支持的调用方式（test_sdk.py main7 示例）：
with InvocationContext(tenant_id="t1", user_id="u1", language="en_US"):
    result = await obj.invoke_action(...)
```

> **注意**：`datacloud-data` 的 `language` 使用连字符格式 `zh-CN`（非下划线 `zh_CN`），
> `normalize_language()` 会自动处理两种格式的互转。

#### 需要补充的：`tool_wrapper.py` 传递 language

**文件**：`packages/datacloud-analysis/src/datacloud_analysis/orchestration/execution/tool_wrapper.py`

当前 `tool_wrapper.py` 构建 `InvocationContext` 时**未传入 `language`**（两处，约 line 680 和 line 740）：

```python
# 当前代码（缺少 language）：
_inv_ctx: Any = InvocationContext(
    user_id=_gc_user_id,
    session_id=_gc_session_id,
    gateway_context=gateway_context,
    workspace_dir=str(workspace_root) if workspace_root is not None else "",
    result_file_storage=_result_file_storage,
    extras=_extras,
)
```

**改动**：从 `configurable` 读取 `locale` 并作为 `language` 传入（两处均需修改）：

```python
# 改后：
_locale = str((config.get("configurable") or {}).get("locale") or "zh_CN")
_inv_ctx: Any = InvocationContext(
    user_id=_gc_user_id,
    session_id=_gc_session_id,
    gateway_context=gateway_context,
    workspace_dir=str(workspace_root) if workspace_root is not None else "",
    result_file_storage=_result_file_storage,
    extras=_extras,
    language=_locale,   # ← 新增，透传 locale 到 datacloud-data SDK
)
```

> `InvocationContext` 的 `language` 参数会经过 `normalize_language()` 处理，
> 因此传入 `zh_CN`（下划线）或 `zh-CN`（连字符）均可正确规范化。

**传递路径**：

```
header_metadata.language = "en_US"
    → worker.py: context.locale = "en_US"
    → graph_input["prompts_overwrite"]["locale"] = "en_US"
    → configurable["locale"] = "en_US"（worker.py line 1449）
    → tool_wrapper.py: InvocationContext(language="en_US")
    → RequestContext.language = "en_US"
    → datacloud-data SDK 内部: get_current_language() → "en_US"
    → 查询结果、错误信息、溢出提示等按英文返回
```

## 5. 完整传递链路（改造后）

```
前端请求
  header_metadata.language = "en_US"
        ↓
byclaw-data/worker.py
  _normalize_locale("en_US") → "en_US"
  context.locale = "en_US"
  graph_input["prompts_overwrite"] = {"locale": "en_US"}
        ↓
datacloud-analysis/graph_builder.py
  overwrite = prompts_overwrite or {}
  locale = overwrite.get("locale") → "en_US"
  get_system_prompt("en_US") → 英文 system prompt
  get_execution_prompt("en_US") → 英文执行规则
        ↓
LLM 以英文 system prompt 推理，返回英文答案
        ↓
用户收到英文回答
```
