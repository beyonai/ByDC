## 核心修复：中间件接口升级

### 问题
原始实现的三个中间件使用自定义接口（`process()` / `get_tools()`），不兼容 DeepAgents SDK 的 `AgentMiddleware` 协议，导致以下错误：
```
AttributeError: type object 'KnowledgeInjectionMiddleware' has no attribute 'wrap_tool_call'
```

### 修复
将三个中间件改为继承 `AgentMiddleware`（来自 `langchain.agents.middleware.types`）：

| 中间件 | 旧接口 | 新接口 |
|--------|--------|--------|
| KnowledgeInjectionMiddleware | `process()` | `awrap_model_call()` |
| DatacloudOutputMiddleware | `get_tools()` | `self.tools = [emit_result]` |
| WorkspaceInitMiddleware | `process()` | `wrap_model_call()` / `awrap_model_call()` |

---

## 中间件实现详情

### KnowledgeInjectionMiddleware
- 继承 `AgentMiddleware`
- 实现 `awrap_model_call()` 拦截每次模型调用
- 从 `request.messages` 提取最后一条用户消息作为检索词
- 调用 `search_knowledge.ainvoke()` 检索本体知识
- 用 `append_to_system_message()` 注入 `<ontology_context>` 到系统消息

### DatacloudOutputMiddleware
- 继承 `AgentMiddleware`
- `self.tools = [emit_result]` 提供工具给 Agent
- `emit_result` 工具将结果转换为 6001 协议格式，支持 4 种类型

### WorkspaceInitMiddleware
- 继承 `AgentMiddleware`
- 实现 `wrap_model_call()` / `awrap_model_call()`
- 首次模型调用时注入工作区路径和 Agent 名称（只注入一次）
