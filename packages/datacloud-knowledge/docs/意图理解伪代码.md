## 算法 A：多槽位语义解析（Slot Filling）

**输入**：用户自然语言问题 $Q$；
**输出**：结构化槽位集合 $S$。

1. **初始化**：创建空槽位集合：

   ```
   S = {
       mentions:         [],   # 术语提及（统一走术语库召回）
       sort_semantics:   [],   # 排序意图（方向 + 截断 + 绑定词）
       time_exprs:       [],   # 时间表达（结构化时间区间）
       display_intents:  [],   # 展示意图（可视化类型，不参与查询）
   }
   ```

2. **大模型槽填充**：调用大模型 Structured Output，对 $Q$ 执行 **Mention Detection + Semantic Role Tagging**：

   - **术语提及判定规则**（纳入 `mentions`，标注语义角色 `role`）：

     | Role | 含义 | 判定规则 | 示例 |
     |------|------|---------|------|
     | `ENTITY` | 查询主体 | 实体名称——要查什么"东西" | "企业"、"订单"、"客户" |
     | `METRIC` | 度量/指标 | 业务指标名——要看什么"数值" | "营收"、"利润率"、"人效" |
     | `DIMENSION_VALUE` | 维度取值 | 用于约束范围的值，但此阶段不确定属于哪个维度 | "亦庄"、"北京"、"XX行业" |
     | `QUALIFIER` | 限定修饰 | 对实体或指标的定语修饰，可能展开为条件组合 | "高效能"、"关键"、"典型" |
     | `COMPOUND` | 复合术语 | 修饰语+实体的不可拆分组合，整体作为一个术语 | "高效能企业"、"关键经营指标"、"高危大客户" |

     - **关键约束**：即使词语在术语库中未知，也要提取——本阶段只负责"识别 + 标注角色"，不负责"匹配到 schema"
     - **一切皆术语**：`DIMENSION_VALUE`（如"亦庄"）与 `ENTITY`（如"企业"）走相同的术语库召回路径，schema 映射在算法 B 中完成

   - **排序意图判定规则**（纳入 `sort_semantics`）：
     - 程度极值词（"最好"、"最高"、"最低"、"最大"、"最差"）→ `direction` + `limit`
     - 排名截断词（"前10"、"TOP 5"）→ `direction=DESC` + `limit=N`
     - 若排序词修饰了某个 mention（如"经营效益**最好**"），则记录 `bound_to` 指向该 mention

   - **时间表达判定规则**（纳入 `time_exprs`）：
     - 相对时间（"去年"、"近三年"）→ 结构化为 `{start, end}`
     - 绝对时间（"2025年"、"2025年Q1"）→ 结构化为 `{start, end}`

   - **展示意图判定规则**（纳入 `display_intents`）：
     - 可视化类型（"柱状图"、"趋势图"、"占比"）
     - 分析模式（"对比"、"排名"、"明细"）
     - 展示意图不参与 SQL 构建，仅传递给前端渲染层

3. **槽位依赖绑定**：处理跨槽引用关系：
   - 若 `sort_semantics` 中存在 `bound_to` 指向某个 mention，建立引用链接
   - 若 `COMPOUND` 类型 mention 可拆解为 `QUALIFIER` + `ENTITY`/`METRIC`，保留整体不拆（拆分决策留给算法 B 术语库匹配阶段）

4. **返回**：输出结构化槽位集合 $S$。

---

### 算法 A 推演示例

**输入**：$Q =$ "在亦庄内XX行业里面经营效益最好的企业是哪个"

**执行过程**：

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 初始化 | `S = {mentions: [], sort_semantics: [], time_exprs: [], display_intents: []}` |
| 2 | 大模型槽填充 | 识别出语义片段：{亦庄, XX行业, 企业, 经营效益, 最好} |

| 2-b | 排序意图标注 | "最好"→`{direction: DESC, limit: 1, bound_to: "经营效益"}` |

**输出**：

```json

    {"text": "企业",     "role": "ENTITY"},
    {"text": "经营效益", "role": "METRIC"},
    {"text": "亦庄",     "role": "DIMENSION_VALUE"},
    {"text": "XX行业",   "role": "DIMENSION_VALUE"}

  "sort_semantics": [
    {"direction": "DESC", "limit": 1, "bound_to": "经营效益"}
  ],
  "time_exprs": [],
  "display_intents": []
}
```

## 算法 B：术语匹配与召回

**输入**：术语提及列表 `S.mentions`；用户ID $u$；术语库缓存 $C$（含全局 `vocabulary_set`、全局 `name_index`、用户级 LRU 缓存）；
**输出**：每个提及词的匹配术语集合 $R = \{m_i \rightarrow [candidate_j]\}$。

1. **预热缓存**：采用二级缓存策略：

   **一级缓存（全局 mmap）**：启动时从数据库加载全局术语到内存：
   - 从 `term_vocabulary` 表加载所有词汇构建 `vocabulary_set`（用于分词）
   - 从 `term_name` + `term` 表联查构建 `name_index`（**仅加载 `scope_user_id IS NULL` 的全局别名**）：
     - `name_index` 结构：`{name_text: [(term_id, term_type_code, match_type), ...]}`
     - `match_type` 取值：`standard_name`（标准名）或 `alias`（别名）
   - 使用 mmap 持久化缓存，跨进程共享，启动时快速加载

   **二级缓存（用户级 LRU）**：为每个活跃用户维护轻量 LRU 缓存：
   - 仅存该用户的专属 `TermName` 记录（`scope_user_id = u`）
   - 首次访问时从 DB 加载，后续命中 LRU；用户别名通常仅几十到几百条
   - 结构与 `name_index` 相同，额外携带 `score` 字段

2. **双向最大匹配分词**：对用户问题 $Q$ 执行 Bidirectional Maximum Matching（BIMM）：

   - **正向最大匹配（FMM）**：从左到右，优先匹配最长词
   - **逆向最大匹配（BMM）**：从右到左，优先匹配最长词
   - **合并策略**：选择匹配词数更少的结果；若相同，选择单字词更少的结果
   - 输出：`matched_words = [(word, start_pos, end_pos), ...]`

3. **精确匹配召回**：对 `matched_words` 中每个匹配词 $w$，**合并全局缓存与用户缓存**结果：

   - 查询全局 `name_index`（哈希查找，O(1)），获取全局候选
   - 查询用户 LRU 缓存，获取该用户的专属别名候选
   - **合并策略**：将两级缓存的候选合并；若同一 `term_id` 同时出现在全局和用户缓存中，保留用户缓存版本（携带个人 `score`）
   - 置信度：`conf = 1.0`（标准名）或 `conf = 0.9`（别名）
   - **去重**：同一 `term_id` 只保留首次出现
   - **注**：由于全局缓存只含全局别名、用户缓存只含当前用户别名，scope 过滤在缓存层已隐式完成，无需后续再过滤

4. **模糊匹配补全**：

   - 对未匹配的连续文本片段（长度≥2）执行 RapidFuzz 模糊匹配
   - 使用 `fuzz.WRatio` 综合相似度算法
   - 阈值：`score_cutoff = 50`（相似度≥50%）
   - 返回 Top-5 候选，携带相似度分与编辑距离
 
5. **候选聚合**：合并精确匹配结果，按置信度排序，作为 $m_i$ 的候选集合。

6. **返回**：输出候选映射 $R = \{m_1 \rightarrow [r_1, r_2, ...], m_2 \rightarrow [...], ...\}$。

---

### 算法 B 推演示例

**输入**：
- 用户 $u =$ "user_001"
- 术语库缓存状态：
  - **全局 mmap 缓存**：
    - `vocabulary_set` = {"企业", "经营", "效益", "亦庄", "行业", ...}
    - `name_index` = {
      "企业": [(TERM_001, "OBJECT", "standard_name")],
      "亦庄": [(TERM_005, "DIMENSION", "alias")],
      "行业": [(TERM_006, "DIMENSION", "standard_name")],
      ...
    }
  - **用户 LRU 缓存**（user_001）：无个人别名记录（空）

**执行过程**：

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 预热缓存 | 加载全局 mmap（vocabulary_set + name_index，约 2ms）+ 加载 user_001 的 LRU 缓存（无记录） |
| 2 | 双向最大匹配 | matched_words = [("企业", 0, 2), ("亦庄", 3, 5), ("行业", 7, 9), ...] |
| 3 | 精确匹配召回 | 全局 mmap："企业"→TERM_001，"亦庄"→TERM_005，"行业"→TERM_006；用户 LRU：无命中；merge 后结果不变 |
| 4 | 模糊匹配补全 | "经营效益"未精确匹配 → RapidFuzz 返回相似候选（仅展示） |

**输出**：
- 精确匹配：$R = \{"企业" \rightarrow [(TERM\_001, 1.0)], "亦庄" \rightarrow [(TERM\_005, 0.9)], "行业" \rightarrow [(TERM\_006, 1.0)]\}$
- 模糊推荐（展示）：`[{span: "经营效益", matches: [(TERM_002, 0.72, edit_dist=2), ...]}]`

**实现说明**：

1. 模糊匹配结果仅用于前端展示推荐，不参与图谱查询的实体匹配
2. 缓存采用二级策略：全局 mmap（跨进程共享）+ 用户级 LRU（轻量快速）
3. scope 过滤在缓存层隐式完成：全局 mmap 只含全局别名，用户 LRU 只含当前用户别名，合并后结果天然不含其他用户的私有别名

## 算法 C：多维消歧

**输入**：候选术语集合 $R$；术语关系图谱 $G$；
**输出**：确权术语列表 $CONF$，歧义术语列表 $AMB$。

1. **层1：score加权重排**

   对候选记录，计算综合得分：
   $$final\_score = conf \times (1 + 0.5 \times score)$$
   
   其中：
   - $conf$：召回渠道基础置信度（精确=1.0，模糊=归一化相似度）
   - $score$：个人专属权重（全局别名默认为0，用户别名携带个人 score，已在算法 B 召回时带入）
   
   按综合得分降序排列，取 Top-3 进入下一层。

2. **层2：图谱拓扑校验**

   若本轮问题中存在**已确权的锚定术语 $A$**（第一级精确匹配命中的词）：
   - 以 $A$ 为起点，在术语关系图谱 $G$ 中 BFS 游走，计算拓扑距离 $d(A, r_j)$
   - 对候选 $r_j$ 按拓扑距离加权：距离越小权重越高
   - 若 $r_j$ 与 $A$ 无连通路径 → 降低权重或删除
   
   注：若无锚定术语，跳过本层。

3. **返回**：输出确权列表 $CONF$ 和歧义列表 $AMB$。

---

### 算法 C 推演示例

**输入**：
- $R = \{"企业" \rightarrow [(TERM\_001, 1.0)], "经营效益" \rightarrow [(TERM\_002, 0.72), (TERM\_003, 0.68), (TERM\_004, 0.45)], "亦庄" \rightarrow [(TERM\_005, 1.0)], "XX行业" \rightarrow [(TERM\_006, 0.85)]\}$
- $G$：TERM_001（企业大宽表）与 TERM_002（净利润）、TERM_003（营收增长率）均有直接关联

**执行过程**：

| 层 | 操作 | 结果 |
|----|------|------|
| 层1 | score重排 | "经营效益"候选排序：TERM_002(0.72) > TERM_003(0.68) > TERM_004(0.45) |
| 层2 | 图谱校验 | 锚定术语A=TERM_001；TERM_002/TERM_003均与TERM_001连通，保留；TERM_004无连通，降低权重 |

**输出**：
- $CONF = [{"user\_word": "企业", "term\_id": "TERM\_001", "term\_name": "企业大宽表"}, {"user\_word": "亦庄", "term\_id": "TERM\_005", "term\_name": "区域"}, {"user\_word": "XX行业", "term\_id": "TERM\_006", "term\_name": "行业分类"}]$
- $AMB = [{"user\_word": "经营效益"}]$

## 算法 D：追问澄清判定

**输入**：歧义术语列表 $AMB$；确权术语列表 $CONF$；原始问题 $Q$；
**输出**：追问状态 $STATE$；追问内容 $MSG$；用户回复后的处理结果 $RESULT$。

1. **追问类型判定**：对 $AMB$ 中每个歧义词 $a$，判定追问策略：

   | 情况 | 条件 | 追问策略 |
   |------|------|---------|
   | 多义竞争 | 候选集中有2~3个术语语境同等合理 | 提供选项让用户选择 |
   | 完全未知 | 无任何候选命中 | 引导用户给出定义、解释 |
   | 部分未知 | 有候选但置信度过低 | 提示候选列表+允许自定义 |

2. **追问内容生成**：基于已确权的术语 $CONF$ 和歧义词 $AMB$，生成追问消息：

   ```
   状态：NEEDS_CLARIFICATION
   
   我已锁定【{CONF中术语名}】作为查询对象。
   但还有几个概念需要您定义：
   {对每个歧义词，按追问策略生成提问}
   ```

3. **等待用户回复**：挂起当前流程，等待用户输入澄清内容 $reply$。

4. **存储处理**：

   - 若用户选择已有术语 → 新建 `TermName`，打上 `scope_user_id`，初始化：$score=1.0, use\_count=1, confirmed\_count=1$
   - 若用户选择自定义，并给出解释 → 新建 `Term`（类型为通用），写入 `TermKnowledge`

5. **返回**：输出追问状态和处理结果。

**未来考虑（当前忽略）**：
让自定义术语成为图谱中的一等公民——有 Term 实体、有明确的 term_type_code、有 TermRelation 边连接到已有术语——而不是塞进 TermKnowledge 当文本。这样它才能：
1. 参与算法 C 的 BFS 拓扑校验（_bfs_cte_sql 通过 term_relation 遍历）
2. 参与 N 跳子图查询（query_n_hop_subgraph / _batch_query_subgraphs）
3. 被 name_index 精确匹配命中

## 算法 E：score闭环更新

**输入**：本轮对话使用的别名记录列表 $A$；用户ID $u$；对话成功标志 $success$；
**输出**：更新后的 $name\_tags.score$。

1. **时间衰减因子计算**：

   对每条别名记录 $a$，计算距上次使用的天数 $days$：
   
   $$decay\_factor = 
   \begin{cases} 
   1.0 & days \leq 30 \\
   0.8 & 31 \leq days \leq 90 \\
   0.5 & days > 90
   \end{cases}$$

2. **计数更新**：

   - 若 $success = true$（别名被LLM确认）：
     - $use\_count \leftarrow use\_count + 1$
     - $confirmed\_count \leftarrow confirmed\_count + 1$
   
   - 若 $success = false$（别名被LLM淘汰）：
     - $use\_count \leftarrow use\_count + 1$
     - $confirmed\_count$ 不变

3. **score重算**：

   $$score = \frac{confirmed\_count}{use\_count + 1} \times decay\_factor$$

4. **时间戳更新**：

   $$last\_used\_at \leftarrow 当前时间戳$$

5. **异步写库**：将更新后的 $name\_tags$ 写入 `TermName` 表，不阻塞主流程。

6. **返回**：完成score更新。

---

### 算法 E 推演示例

**输入**：
- 用户 $u =$ "user_001" 在本轮对话中使用别名"公司"→TERM_001
- 原始 $name\_tags$：
  ```json
  {
    "scope_user_id": "user_001",
    "score": 0.93,
    "use_count": 28,
    "confirmed_count": 26,
    "last_used_at": "2026-03-28T09:30:00Z"
  }
  ```
- $success = true$（LLM确认了该别名）
- 当前时间：2026-03-28T10:00:00Z

**执行过程**：

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 时间衰减 | $days = 0.5小时 \leq 30天$ → $decay\_factor = 1.0$ |
| 2 | 计数更新 | $use\_count = 28 + 1 = 29$，$confirmed\_count = 26 + 1 = 27$ |
| 3 | score重算 | $score = \frac{27}{29+1} \times 1.0 = 0.90$ |
| 4 | 时间戳更新 | $last\_used\_at = 2026-03-28T10:00:00Z$ |

**输出**：
```json
{
  "scope_user_id": "user_001",
  "score": 0.90,
  "use_count": 29,
  "confirmed_count": 27,
  "last_used_at": "2026-03-28T10:00:00Z"
}
```

## 完整流程推演示例

**用户问题**："帮我看一下有哪些企业属于高效能企业，并展示其中一个典型企业关键的经营指标"
**用户**：user_001（首次使用，无个人别名记录）

### Step 1：多槽位语义解析（算法 A）

**输入**：Q = "帮我看一下有哪些企业属于高效能企业，并展示其中一个典型企业关键的经营指标"

**执行过程**：

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 初始化 | S = {mentions: [], sort_semantics: [], time_exprs: [], display_intents: []} |
| 2 | 大模型槽填充 | 识别语义片段：{企业, 高效能企业, 典型企业, 关键经营指标, 展示} |
| 2-a | 术语提及标注 | "企业"→ENTITY；"高效能企业"→COMPOUND；"典型企业"→COMPOUND；"关键经营指标"→COMPOUND |
| 2-b | 排序意图标注 | 无排序词检出 |
| 2-c | 时间表达标注 | 无时间表达检出 |
| 2-d | 展示意图标注 | "展示"→display_intent |
| 3 | 槽位依赖绑定 | COMPOUND 类型保留整体不拆（拆分决策留给算法 B） |

**输出**：

## 算法复杂度分析

| 算法 | 时间复杂度 | 空间复杂度 | 备注 |
|------|------------|------------|------|
| A: 多槽位语义解析 | $O(1)$ LLM调用 | $O(\|S\|)$ | 单次大模型推理，输出结构化槽位 |
| B: 多路召回 | $O(\|M\| \times K)$ | $O(\|M\| \times K)$ | 并行召回，M=mentions数，K=Top-K候选数 |
|| C: 多维消歧 | $O(\|M\| \times 2 + \|G\|)$ | $O(\|M\|)$ | score重排 + BFS图谱遍历（2层漏斗） |
| D: 追问澄清 | $O(\|AMB\|)$ | $O(\|AMB\|)$ | 单次追问生成 |
| E: score更新 | $O(\|A\|)$ | $O(1)$ | 异步写库，不阻塞 |

---

## 总结

本方案通过五个独立算法形成完整的意图理解闭环：

1. **算法A**：多槽位语义解析——识别术语提及并标注语义角色（一切皆术语，不做 schema 映射）
2. **算法B**：多路召回候选术语（精确→模糊，**二级缓存隐式完成 scope 过滤**，所有 mentions 统一走术语库）
3. **算法C**：多维消歧确权（score→图谱两层漏斗，scope 过滤已前置到算法 B）
4. **算法D**：追问澄清并存储用户解释
5. **算法E**：score闭环更新（越用越准的正向循环）

核心理念：**术语库只做锚定与定调，复杂过滤和常识性操作交给大模型自由发挥**。
