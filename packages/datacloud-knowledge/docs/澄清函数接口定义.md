# 澄清函数接口定义

---

## 1. 共享子结构

### 1.1 FilterItem

过滤条件项。新策略下 `field` / `value` 可能是中文业务术语。

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `field` | `str` | 是 | 字段名。可能是中文术语（如 `"营收"`）或已确认的 field_code（如 `"revenue"`） |
| `op` | `str` | 是 | 操作符：`eq` \| `in` \| `gt` \| `gte` \| `lt` \| `lte` \| `between` \| `is_null` \| `is_not_null` |
| `value` | `Any` | 否 | 字面量（数字、日期字符串）、数组、或中文描述（如 `"龙头"`、`"高"`）。`is_null` / `is_not_null` 时可省略 |

```json
{"field": "营收", "op": "gt", "value": 1000000}
{"field": "行业", "op": "in", "value": ["制造业", "农业"]}
{"field": "效能等级", "op": "in", "value": ["中效", "低效"]}
```

### 1.2 OrderByItem

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `field` | `str` | 是 | 字段名（中文术语）或 `metrics.as` 别名（如 `"total_revenue"`） |
| `direction` | `str` | 是 | `"asc"` \| `"desc"` |

### 1.3 MetricItem

统计指标项，五种形态（§1.6.2 + §3.1.3 扩展）。

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `field` | `str` | 否 | 字段名（中文术语），与 `expr` 二选一。`count_all` 时两者均不填 |
| `expr` | `str` | 否 | 公式表达式（如 `"营收 - 成本"`），与 `field` 二选一 |
| `agg` | `str` | **是** | 聚合函数：`sum` \| `avg` \| `max` \| `min` \| `count` \| `count_all` |
| `as` | `str` | **是** | 输出别名（如 `"total_revenue"`） |
| `filters` | `list[FilterItem]` | 否 | 条件聚合过滤（生成 CASE WHEN），与 `field`/`expr` 正交可叠加 |

五种形态示例：

```json
// ① count_all
{"agg": "count_all", "as": "enterprise_count"}

// ② 普通字段聚合
{"field": "营收", "agg": "sum", "as": "total_revenue"}

// ③ 条件字段聚合
{"field": "营收", "filters": [{"field": "企业级别", "op": "eq", "value": "龙头"}], "agg": "sum", "as": "leading_revenue"}

// ④ 公式聚合
{"expr": "营收 - 成本", "agg": "avg", "as": "avg_profit"}

// ⑤ 条件公式聚合
{"expr": "营收 - 成本", "filters": [{"field": "企业级别", "op": "in", "value": ["龙头", "骨干"]}], "agg": "sum", "as": "key_enterprise_profit"}
```

### 1.4 DimensionItem

分组维度项。`dimensions` 数组中的元素可以是**字符串简写**或**对象形式**。

字符串简写（等价于 `group_op: "self"`）：

```json
"行业"
```

对象形式：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `field` | `str` | 是 | 字段名（中文术语） |
| `group_op` | `str` | 是 | `"self"` \| `"year"` \| `"month"` \| `"quarter"` \| `"day"` \| `"range"` |
| `buckets` | `list[BucketItem]` | 否 | `group_op="range"` 时必填 |

BucketItem：

| 字段 | 类型 | 说明 |
|------|------|------|
| `from` | `number \| null` | 下界（含），`null` 表示无下界 |
| `to` | `number \| null` | 上界（不含），`null` 表示无上界 |
| `label` | `str` | 分桶标签 |

```json
{"field": "创建月份", "group_op": "month"}
{"field": "营收", "group_op": "range", "buckets": [
  {"from": null, "to": 500000, "label": "低收入"},
  {"from": 500000, "to": 2000000, "label": "中收入"},
  {"from": 2000000, "to": null, "label": "高收入"}
]}
```

### 1.5 HavingItem

聚合后过滤项。`field` 必须引用 `metrics` 中某项的 `as` 别名。

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `field` | `str` | 是 | `metrics.as` 别名（如 `"total_revenue"`） |
| `op` | `str` | 是 | `eq` \| `gt` \| `gte` \| `lt` \| `lte` |
| `value` | `Any` | 是 | 阈值 |

## 2. StructuredQuery

`query_{code}` 工具的完整参数结构，即 LLM 调用工具时填写的完整 JSON。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `query` | `str` | **是** | — | 用户原始问题的完整自然语言描述 |
| `complex_conditions` | `list[str]` | 否 | `[]` | 溢出过滤区：无法用字面常量表达的过滤条件 |
| `select` | `list[str]` | 否 | — | 返回字段列表（中文术语） |
| `filters` | `list[FilterItem]` | 否 | — | 过滤条件 |
| `filter_relation` | `str` | 否 | `"AND"` | `"AND"` \| `"OR"` |
| `order_by` | `list[OrderByItem]` | 否 | — | 排序规则 |
| `limit` | `int` | 否 | `100` | 返回行数上限 [1, 1000] |
| `offset` | `int` | 否 | `0` | 分页偏移量 ≥ 0 |

完整示例：

```json
{
  "query": "查询本月新增的、营收大于100万的制造业或农业企业，按营收降序取前20条",
  "select": ["企业名称", "营收", "创建日期", "行业"],
  "filters": [
    {"field": "创建日期", "op": "between", "value": ["2026-04-01", "2026-04-30"]},
    {"field": "营收",     "op": "gt",      "value": 1000000},
    {"field": "行业",     "op": "in",      "value": ["制造业", "农业"]}
  ],
  "filter_relation": "AND",
  "order_by": [{"field": "营收", "direction": "desc"}],
  "limit": 20,
  "offset": 0,
  "complex_conditions": []
}
```

## 3. StructuredCompute

`compute_{code}` 工具的完整参数结构，即 LLM 调用工具时填写的完整 JSON。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `query` | `str` | **是** | — | 用户原始问题的完整自然语言描述 |
| `complex_conditions` | `list[str]` | 否 | `[]` | 溢出过滤区 |
| `dimensions` | `list[str \| DimensionItem]` | 否 | — | 分组维度（字符串简写或对象形式） |
| `metrics` | `list[MetricItem]` | **是** | — | 统计指标（至少一项） |
| `filters` | `list[FilterItem]` | 否 | — | 行级过滤（WHERE） |
| `having` | `list[HavingItem]` | 否 | — | 聚合后过滤（HAVING） |
| `filter_relation` | `str` | 否 | `"AND"` | `"AND"` \| `"OR"` |
| `order_by` | `list[OrderByItem]` | 否 | — | 排序 |
| `limit` | `int` | 否 | `100` | [1, 1000] |

完整示例（覆盖 MetricItem 五种形态 + DimensionItem 对象形式）：

```json
{
  "query": "按行业和地区统计平均利润及龙头骨干企业总利润，过滤一季度数据，按平均利润降序",
  "dimensions": ["行业", "地区"],
  "metrics": [
    {"agg": "count_all", "as": "enterprise_count"},
    {"field": "营收", "agg": "sum", "as": "total_revenue"},
    {"expr": "营收 - 成本", "agg": "avg", "as": "avg_profit"},
    {
      "expr": "营收 - 成本",
      "filters": [{"field": "企业级别", "op": "in", "value": ["龙头", "骨干"]}],
      "agg": "sum",
      "as": "key_enterprise_profit"
    }
  ],
  "filters": [
    {"field": "创建日期", "op": "between", "value": ["2026-01-01", "2026-03-31"]}
  ],
  "having": [
    {"field": "total_revenue", "op": "gt", "value": 1000000}
  ],
  "order_by": [{"field": "avg_profit", "direction": "desc"}],
  "limit": 50,
  "complex_conditions": []
}
```

### 4.1 analyze_query_clarification_query

查询类工具（`query_*` / `data_query_*`）的歧义分析。

```python
def analyze_query_clarification_query(
    query: str,
    ontology_code: str,
    structured_query: dict,
) -> ClarificationResult:
```

**入参：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `query` | `str` | 用户原始问题（从 `structured_query["query"]` 提取，与 dict 中的值相同） |
| `ontology_code` | `str` | 对象编码，从工具名提取（`query_Enterprise` → `"Enterprise"`） |
| `structured_query` | `dict` | 完整的 StructuredQuery（见 §2），包含 `query`、`complex_conditions` 及所有业务参数 |

**出参：**

### 4.2 analyze_query_clarification_compute

统计类工具（`compute_*`）的歧义分析。

```python
def analyze_query_clarification_compute(
    query: str,
    ontology_code: str,
    structured_compute: dict,
) -> ClarificationResult:
```

**入参：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `query` | `str` | 用户原始问题 |
| `ontology_code` | `str` | 对象编码（`compute_Enterprise` → `"Enterprise"`） |
| `structured_compute` | `dict` | 完整的 StructuredCompute（见 §3），包含 `query`、`complex_conditions` 及所有业务参数 |

**出参：** 
**注意：** `structured_compute` 中需要扫描歧义的位置比 query 多：

- `filters[].field` / `filters[].value`（同 query）
- `dimensions[]`（字符串简写或 `DimensionItem.field`）
- `metrics[].field` / `metrics[].expr` 中的术语
- `metrics[].filters[].field` / `metrics[].filters[].value`（条件聚合内嵌的过滤）

### 4.3 format_clarification_query

将用户的澄清选择应用到查询参数，返回可直接传给 `query_{code}` 的更新后参数。

```python
def format_clarification_query(
    structured_query: dict,
    form: dict,
) -> dict:
```

**入参：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `structured_query` | `dict` | 完整的 StructuredQuery（与传给 `analyze_query_clarification_query` 的相同） |
| `form` | `dict` | 前端用户选择后回传的表单数据（即 `interrupt` 后的 `resume_value`） |

**`form` 结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `paradigmList` | `list[ParadigmItem]` | 用户从候选中选择/确认的条目 |


**出参：** `dict`，更新后的 StructuredQuery，中文术语已替换为确认的 field_code / 枚举 code。

**出参结构与 StructuredQuery 相同**，但所有歧义术语已被替换：

```json
{
  "query": "查询本月新增的、营收大于100万的制造业或农业企业，按营收降序取前20条",
  "select": ["enterprise_name", "revenue", "created_date", "industry"],
  "filters": [
    {"field": "created_date", "op": "between", "value": ["2026-04-01", "2026-04-30"]},
    {"field": "revenue",      "op": "gt",      "value": 1000000},
    {"field": "industry",     "op": "in",      "value": ["manufacturing", "agriculture"]}
  ],
  "filter_relation": "AND",
  "order_by": [{"field": "revenue", "direction": "desc"}],
  "limit": 20,
  "offset": 0,
  "complex_conditions": []
}
```

**核心职责：**

1. 遍历 `structured_query` 中所有包含中文的位置（`select`、`filters[].field`、`filters[].value`、`order_by[].field`）
2. 根据 `form.paradigmList` 中用户确认的映射，将中文替换为 标准中文名称
3. 返回替换后的完整 dict，确保 检测全部命中 1:1 → CLEAR

### 4.4 format_clarification_compute

将用户的澄清选择应用到统计参数，返回可直接传给 `compute_{code}` 的更新后参数。

```python
def format_clarification_compute(
    structured_compute: dict,
    form: dict,
) -> dict:
```

**入参：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `structured_compute` | `dict` | 完整的 StructuredCompute（与传给 `analyze_query_clarification_compute` 的相同） |
| `form` | `dict` | 前端回传的表单数据（结构同 §4.3 的 `form`） |

**出参：** 
**核心职责：**

1. 遍历 `structured_compute` 中所有包含中文术语的位置：
   - `dimensions[]`（字符串简写和 `DimensionItem.field`）
   - `metrics[].field`
   - `metrics[].expr` 中的术语
   - `metrics[].filters[].field` / `metrics[].filters[].value`
   - `filters[].field` / `filters[].value`
   - `order_by[].field`（非 `metrics.as` 别名的部分）
2. 根据 `form.paradigmList` 中用户确认的映射替换
3. 返回替换后的完整 dict
