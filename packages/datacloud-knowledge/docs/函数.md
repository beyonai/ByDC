# 函数.md
## 函数 1
* 分库设计，一个库，对应一个图: 
    * 新增GraphManager类
* 编辑术语：
    * 新增&更新：节点、节点属性、关系
    * 删除：节点、节点属性、关系
* 术语添加标签：
    * 新增&更新
    * 删除
    * 查询
* 编辑本体，本体是树，按照树的增删改来,参考`examples/crm/data/scenes/main.json`
* 给一个根节点本体，查询整个树：
> ViewNode/Properties/ObjectRelationNode/ObjectNode/FieldNode/ActionNode/ParameterNode/FunctionNode等具有严格的层级关系，参考`examples/crm/data/scenes/main.json`

* 从文件导入本体、术语（注：批量编辑操作用文件）：
    * 使用GraphManager类导入导出JSON文件
    * 暂不支持JSON外的任何形式

## 函数 2
* 术语标准化查询：
    * 传入标准名、关联的TermType
    * 只返回精准匹配结果
    * 匹配不上，按拼音、编辑距离返回推荐结果

## 函数 3
* 根据元数据检索文档:
    * 传入自然语言以及元数据（标签）条件
    * 返回文档术语节点
    * 文档术语的属性有个kv对内v是整篇文档
    * 文档术语的属性有个kv对内v是文件ID
    * 文档切片向量化，传入文件ID列表召回，召回后提取归属的文档ID

    ```
class Operator(str, Enum):
    """支持的查询操作符"""
    EQ = "eq"           # 等于
    NE = "ne"           # 不等于  
    GT = "gt"           # 大于
    GTE = "gte"         # 大于等于
    LT = "lt"           # 小于
    LTE = "lte"         # 小于等于
    IN = "in"           # 在列表中
    NIN = "nin"       # 不在列表中
    BETWEEN = "between" # 在范围之间

class LogicalOperator(str, Enum):
    AND = "and"
    OR = "or"
    NOT = "not"
    ```
